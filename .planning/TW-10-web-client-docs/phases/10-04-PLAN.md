# 10-04 — Web Client Feature Parity (Market, StarDock, Combat)

## Objective
Implement the three core gameplay overlays missing from the web client: Market trading, StarDock upgrades, and Combat. Bring the web client to feature parity with the CLI cloud mode.

## Context
The web client scaffold has placeholder views for Market, StarDock, and Combat. The Sector view lacks NPC display, news ticker, and combat triggers. The API endpoints (`/api/action/trade`, `/api/action/upgrade`, `/api/action/combat`) are already live and tested.

## Tasks

### Market Overlay
1. **MarketView.vue** — Full rebuild:
   - Fetch sector inventory on mount (`cloudGetSector`)
   - Display commodity prices (ore/organics/equipment) with buy/sell prices
   - Show supply, owned cargo, profit indicators (▲/▼ vs base price)
   - Quantity selector with +/- buttons
   - Total cost/revenue display
   - Buy/Sell mode toggle
   - Execute trade via `cloudTrade()` API
   - Update ship cargo and credits after trade
   - Refresh sector inventory after trade
   - Keyboard: ↑↓ select, B/S toggle mode, +/- quantity, Enter confirm, Esc back

2. **SectorView.vue** — Wire Market trigger:
   - Clicking "Open Market" button or pressing `M` navigates to MarketView
   - Only show button if current sector has a port

### StarDock Overlay
3. **StarDockView.vue** — Full rebuild:
   - Fetch ship data (upgrades already in ship store)
   - Display owned upgrades list
   - Display available upgrades (next tier only, filtered by prerequisite)
   - Show upgrade cost, description, stat bonuses
   - Purchase button with credit validation
   - Execute purchase via `cloudUpgrade()` API
   - Update ship stats after purchase
   - Keyboard: ↑↓ select, Enter purchase, Esc back

4. **SectorView.vue** — Wire StarDock trigger:
   - Clicking "Enter StarDock" or pressing `D` navigates to StarDockView
   - Only show button if current sector is a stardock

### Combat Overlay
5. **CombatView.vue** — Full rebuild:
   - Display enemy info (name, type, hull, shield)
   - Display player ship stats
   - Action buttons: Attack, Flee, Bribe
   - Execute combat via `cloudCombat()` API
   - Display narrative text from API response
   - Handle all outcomes: win, flee, bribe, destroyed
   - Update ship state after combat (hull, shield, credits, cargo, sector if destroyed)
   - Remove dead NPC from local state
   - Keyboard: ↑↓ select action, Enter confirm

6. **SectorView.vue** — Wire Combat trigger:
   - Display NPCs in sector (raiders in red, patrols in green, traders in cyan)
   - Click NPC to initiate combat
   - Or add "Attack" button per NPC row

### Sector View Polish
7. **SectorView.vue enhancements**:
   - Add NPC list display (max 3 visible, "… and N more" overflow)
   - Add news ticker at bottom (last 2 items)
   - Display port inventory preview when docked (price/supply)
   - Show turn warning when low (yellow ≤20, red = 0)

## Verification
- [ ] Register → create ship → jump to port sector → open Market → buy/sell commodities
- [ ] Jump to stardock sector → open StarDock → purchase upgrade → stats increase
- [ ] Jump to sector with NPC → attack NPC → see combat narrative → win/lose/flee
- [ ] All keyboard shortcuts work (M, D, N, H, L, Esc, ↑↓, Enter)
- [ ] Ship state updates correctly after each action
- [ ] No console errors

- Market, StarDock, and Combat are fully functional in the browser
- Web client reaches feature parity with CLI cloud mode
- All API endpoints tested end-to-end
