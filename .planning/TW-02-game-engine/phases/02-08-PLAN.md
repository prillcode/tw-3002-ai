# 02-08-PLAN — Integration & Test

## Objective
Wire all engine systems together into a cohesive, playable solo experience. Fix balance issues, smooth rough edges, and declare the local single-player MVP complete. This is the capstone phase of TW-02 before handing off to TW-03 (NPC Brain).

## Context
- All engine phases 1-7 implemented (or planned)
- CLI has full UI for navigation, trading, upgrades, combat
- Individual systems tested in isolation but not end-to-end
- No balance pass has happened yet — prices, damage, upgrade costs are rough estimates
- Need to verify the full loop: login → trade → combat → upgrade → save → logout → continue

## Approach

This phase is primarily integration work, balance tuning, and bug fixing — not new features. The goal is to make the existing systems feel like a real game.

## Tasks

### 1. End-to-end playtest checklist

Play through the entire game flow manually, noting issues:

**New Game Flow:**
- [ ] Launch → title screen → new game → slot select → galaxy size → name ship → pick class
- [ ] Verify starting stats match class (cargo, hull, turns)
- [ ] Verify starting sector is in FedSpace
- [ ] Verify shield = max on spawn

**Navigation Loop:**
- [ ] Jump between 10+ sectors
- [ ] Verify turn decrements correctly
- [ ] Verify no turn deficit (can't go below 0)
- [ ] Verify dangerous sectors trigger encounters (not every time, but often enough)
- [ ] Verify safe sectors never trigger combat
- [ ] Verify StarDock sectors accessible and marked

**Trading Loop:**
- [ ] Buy ore at one port
- [ ] Jump to different port
- [ ] Sell ore for profit
- [ ] Verify cargo limits enforced
- [ ] Verify credit limits enforced
- [ ] Verify port stock decreases after buy
- [ ] Verify 3 commodities all tradeable

**Combat Loop:**
- [ ] Enter dangerous sector → encounter triggers
- [ ] Win a fight via attack
- [ ] Flee successfully
- [ ] Flee and fail (get hit)
- [ ] Bribe an enemy
- [ ] Get destroyed → respawn in FedSpace
- [ ] Verify shield regenerates after combat
- [ ] Verify hull does NOT regenerate (must find StarDock or be destroyed)

**Upgrade Loop:**
- [ ] Earn credits through trading
- [ ] Navigate to StarDock sector
- [ ] Buy an upgrade
- [ ] Verify stats update (max cargo increases, etc.)
- [ ] Verify credits deducted
- [ ] Verify tier prerequisites enforced

**Save/Continue Loop:**
- [ ] Quit game mid-session
- [ ] Relaunch → continue → verify exact same state
- [ ] Verify all 3 slots independent
- [ ] Verify overwrite confirmation works

**Edge Cases:**
- [ ] Jump with 0 turns (should be blocked or fail gracefully)
- [ ] Buy when cargo full (blocked)
- [ ] Buy when credits insufficient (blocked)
- [ ] Combat with 0 shield (hull takes full damage)
- [ ] Combat with maxed upgrades (doesn't crash)

### 2. Balance pass

**Economy:**
- Target: a profitable trade loop should take 3-5 jumps and yield 20-50% profit
- Check: buy ore @ 100 → sell @ 150 = good. buy @ 100 → sell @ 110 = too grindy
- Adjust `BASE_PRICES` and port class spread if needed

**Combat:**
- Target: Merchant ship without upgrades should survive 1-2 combats but be at risk in 3+
- Target: Scout should flee successfully ~70% of the time
- Target: Interceptor should win most fights but still take hull damage
- Adjust enemy stats and damage formula if needed

**Upgrade costs:**
- Target: first upgrade affordable after 3-5 profitable trades
- Target: full upgrade set costs ~50,000-80,000 credits (endgame goal)
- Adjust prices in `UPGRADE_CATALOG` if needed

**Turn economy:**
- Target: 80-120 turns should feel like a "session" (15-30 min of play)
- Each jump = 1 turn feels right
- Future: turn regeneration (not in this phase)

### 4. Fix integration bugs

Common issues to watch for:
- [ ] Galaxy not re-generated on new game (reuses old galaxy)
- [ ] Hull damage not persisted in save
- [ ] Shield not reset on new game (carries over from old save)
- [ ] Effective stats not recalculated after upgrade purchase
- [ ] Market prices don't update after trade (need to call `tickEconomy` or refresh)
- [ ] Cargo overflow after upgrade (maxCargo increased but cargo already over new limit — clamp)

### 5. Add a simple score/goal

For MVP completeness, give the player something to aim for:
- Track `netWorth = credits + cargoValue + upgradeValue`
- Show net worth on ship status panel
- Display rank based on net worth:
  - < 10k: Space Peasant
  - 10k-50k: Trader
  - 50k-100k: Merchant Prince
  - 100k+: Galactic Tycoon

This is minimal but gives the loop meaning.

3. Combat feels dangerous but fair

6. README documents current capabilities

## Notes

- **Balance is subjective.** Document your assumptions so TW-03/TW-04 can iterate.
- **If a bug is too big to fix quickly,** document it and move on. Don't let one issue block the whole phase.
- **Score/goal is intentionally minimal.** A real progression system (corporations, missions, leaderboards) is TW-04 scope.
