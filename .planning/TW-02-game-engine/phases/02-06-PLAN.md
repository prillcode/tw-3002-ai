# 02-06-PLAN — Combat System

## Objective
Build the combat system so that dangerous sectors actually matter. Players (and eventually NPCs) can initiate fights, flee, or bribe attackers. Combat resolves in turn-based rounds with damage calculated from weapons vs shields/hull. Ship upgrades (weapons, shields, hull plating) bought at StarDocks finally have gameplay impact.

## Context
- See `../BRIEF.md` for TW-02 scope
- See `../ROADMAP.md` Phase 6
- PRD Section 4.2 mentions combat actions (attack, flee, etc.)
- Existing engine: `packages/engine/src/ships/upgrades.ts` has weapon/shield/hull upgrade definitions
- Existing CLI: `SectorScreen.tsx` shows danger levels but they're purely cosmetic
- TW2002 inspiration: combat is dangerous, flee is often the smart choice, bribe is a credit-based escape

## Approach

Combat lives entirely in the engine (`packages/engine/src/combat/`). The CLI gets a `CombatScreen.tsx` that renders the encounter. Combat is PvE-only for now (no player-vs-player).

### Combat Types
- **Ambush:** Player enters a dangerous sector and an NPC attacks
- **Player-initiated:** Player chooses to attack a known NPC in sector (future, NPCs don't exist yet)
- **For now:** Random encounter chance when entering dangerous sectors. Safe sectors = zero combat.

### Combat Flow
```
Enter dangerous sector → roll encounter chance
  → No encounter: normal sector view
  → Encounter: CombatScreen appears

CombatScreen:
  1. Show attacker info (name, ship type, threat level)
  2. Player chooses: [Fight] [Flee] [Bribe]
  3. Resolve round
  4. Repeat until: player wins / player flees / player destroyed / NPC destroyed / bribe accepted
  5. Return to sector (or FedSpace if destroyed)
```

## Tasks

### 1. Add combat types to engine (`packages/engine/src/combat/`)

**`types.ts`:**
```typescript
export interface Combatant {
  name: string;
  hull: number;
  maxHull: number;
  shield: number;
  maxShield: number;
  weaponDamage: number;      // base damage per hit
  dodgeChance: number;       // 0-1, chance to avoid hit
  credits: number;           // for bribe calculations
}

export type CombatAction = 'attack' | 'flee' | 'bribe';

export interface CombatRound {
  round: number;
  playerAction: CombatAction;
  enemyAction: CombatAction;
  playerDamageDealt: number;
  playerDamageTaken: number;
  shieldAbsorbed: number;
  fled: boolean;
  bribeAccepted: boolean;
  bribeAmount: number;
  enemyDestroyed: boolean;
  playerDestroyed: boolean;
}

export interface CombatResult {
  victory: boolean;
  fled: boolean;
  bribed: boolean;
  rounds: CombatRound[];
  totalDamageDealt: number;
  totalDamageTaken: number;
  creditsLost: number;
  creditsGained: number;
  hullRemaining: number;
  shieldRemaining: number;
}
```

**`npcs.ts` — Enemy templates:**
- Pirate Scout: low hull, high dodge, low damage
- Pirate Raider: medium hull, medium dodge, medium damage  
- Pirate Dreadnought: high hull, low dodge, high damage
- Patrol (FedSpace only if player is offender): balanced

Generate enemy stats based on sector danger + some randomness.

**`resolver.ts` — Combat resolution:**
- `initiateCombat(player, enemy): CombatState`
- `resolveRound(state, playerAction): { state: CombatState; round: CombatRound }`
- `canFlee(player, enemy): boolean` — dodge chance + speed bonus
- `bribeAmount(enemy): number` — scales with enemy strength
- `computeResult(state): CombatResult`

**Damage formula:**
```
hitRoll = random() > defender.dodgeChance
if hit:
  rawDamage = attacker.weaponDamage * randomRange(0.8, 1.2)
  shieldAbsorb = min(defender.shield, rawDamage * 0.5)
  defender.shield -= shieldAbsorb
  hullDamage = rawDamage - shieldAbsorb
  defender.hull -= hullDamage
```

**Flee formula:**
```
fleeChance = player.dodgeChance * 100 + (playerSpeed - enemySpeed) * 5
min 10%, max 90%
```

**Bribe formula:**
```
bribe = enemy.maxHull * 15 + randomRange(-20%, +20%)
min 500 cr, max 50% of player credits
```

### 2. Add encounter chance to galaxy/sector logic

In `packages/engine/src/galaxy/generator.ts` or a new `encounters.ts`:
- `rollEncounter(sector, player): Enemy | null`
- Safe sectors: 0% encounter
- Caution sectors: 5-15% encounter
- Dangerous sectors: 20-40% encounter
- FedSpace: 0% unless player is flagged (not implemented yet)

Enemy strength scales with distance from FedSpace.

**States:**
- `intro` — show enemy, wait for action
- `round` — resolve round, show log
- `result` — victory/defeat/fled/bribed summary
- `destroyed` — ship destroyed, respawn in FedSpace

### 4. Wire combat into SectorScreen

- After jump, check `rollEncounter(newSector, playerStats)`
- If encounter, switch to `combat` mode before showing sector
- Pass `onCombatEnd(result)` callback to update ship state
- If destroyed: reset hull, teleport to FedSpace center, lose 10% credits

### 5. Update App.tsx for combat mode

- Add `'combat'` to `AppMode`
- Handle combat → sector transition
- Apply combat result to ship state (hull damage, credit loss/gain)
- Auto-save after combat (existing useEffect handles this)

### 6. Update save system for hull/shield

- `hull` already saved
- Add `shield` to GameState and DB schema
- Migration: `ALTER TABLE saves ADD COLUMN shield INTEGER DEFAULT 0`
- Load shield from save, initialize to `effectiveStats.maxShield` on new game

### 7. Shield regeneration

- After each jump (or on login), shield regenerates to max
- This makes shield a "first hit buffer" that refreshes between encounters
- Hull only repairs at StarDock (or very slowly, not in this phase)

## Verification

1. **Engine** — `bun run typecheck` passes for engine
2. **No encounter in safe** — Jump 20 times in FedSpace, zero combat
3. **Encounter in dangerous** — Jump 20 times in dangerous sector, see encounters
4. **Combat rounds** — Attack deals damage within expected range
5. **Flee works** — Flee succeeds sometimes, fails sometimes, feels fair
6. **Bribe works** — Credits deducted, combat ends, no damage taken
7. **Destroyed respawn** — Hull 0 → teleport to FedSpace, credits reduced, hull restored
8. **Shield absorbs** — First hits reduce shield before hull
9. **Upgrades matter** — More weapon damage = faster kills. More shields = less hull damage.
10. **Save/load** — Post-combat hull/shield persisted, restored on continue
11. **CLI** — `bun run typecheck` + `bun run build` clean

## Done When
- Entering dangerous sectors can trigger combat encounters
- Combat screen shows enemy info, round-by-round log
- Player can attack, flee, or bribe
- Damage calculated from weapons vs shields/hull
- Shield regenerates between encounters
- Ship destruction → respawn in FedSpace with penalties
- All combat state persisted in save/load
- Danger levels feel meaningful (safe = peaceful, dangerous = risky)
- TypeScript clean, binary builds

## Notes
- **Scope guard:** No NPC AI yet — enemies pick actions randomly (weighted by type). Aggressive enemies favor attack, cowardly favor flee. NPC brain (TW-03) will replace this.
- **No loot yet:** Victory gives small credit reward + bragging rights. Real loot from TW-03.
- **Combat balance:** Start conservative — combat should feel dangerous. Tune after playtesting.
- **Sound effects:** Not possible in terminal, but use visual cues (color flashes, bold text).
