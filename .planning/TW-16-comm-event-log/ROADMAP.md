# TW-16 Roadmap: Comm & Event Log

**Estimated Total:** 3–4 hours (1 focused session)

## Phase 1: Event Log Store & Types (1 hour)
**Goal:** Pinia store with typed entries, consumed by existing views.

**Deliverables:**
- `web/game/src/stores/eventLog.ts` — new Pinia store
- `web/game/src/types/events.ts` — event type definitions
- Event types with tags and icons:

| Tag | Icon | Color | Generated By |
|-----|------|-------|-------------|
| NAV | 🧭 | Cyan | Warp, move |
| TRD | 💰 | Yellow | Buy, sell |
| CMB | ⚔ | Red | Attack, flee, bribe, victory, defeat |
| UPG | ⚡ | Magenta | Hull, shield, cargo, weapon upgrade |
| EVT | 📡 | White | News, fighters deployed/recalled, insurance |
| SYS | 🔧 | Muted | Turn regen, session start, errors |

- Entry shape: `{ id, tag, icon, color, text, sector, timestamp }`
- Store keeps last 100 entries, `addEvent()` helper
- Remove `message` ref from `ship.ts`

**Success:** Can `addEvent('CMB', 'Victory over Pirate Raider +340 cr')` from any view.

---

## Phase 2: Wire All Actions to Event Log (1–2 hours)
**Goal:** Every game action creates a log entry.

**Deliverables:**
- Wire sector warp → `addEvent('NAV', 'Warped to Sector 42')`
- Wire trade → `addEvent('TRD', 'Sold 50 ore for 12,500 cr')`
- Wire combat → `addEvent('CMB', 'Victory over Pirate Raider +340 cr')`
- Wire upgrade → `addEvent('UPG', 'Purchased Hull Plating Mk II')`
- Wire insurance → `addEvent('EVT', 'Insurance activated (7 days)')`
- Wire fighter deploy/recall → `addEvent('EVT', 'Deployed 50 fighters (Defensive)')`
- Wire turn regen → `addEvent('SYS', '+15 turns regenerated')`
- Wire news items → `addEvent('EVT', news.headline)`
- Wire digest deaths → `addEvent('CMB', 'Destroyed by DarkReaver in Sector 42')`
- Remove all `ship.message = ...` assignments across views
- Remove message banner from SectorView

**Success:** Playing the game for 5 minutes produces 15–30 log entries.

---

## Phase 3: Event Log UI (1 hour)
**Goal:** Visible, accessible event log in the game client.

**Deliverables:**
- Replace NavigationView with ActivityLogView (or extend it)
- Shows categorized entries with colored tags and timestamps
- Auto-scrolls to bottom, scrollback for history
- SectorView: mini event ticker showing last 3 entries above warp lanes
- Remove `ship.message` banner entirely
- Clicking an entry with a sector number could navigate there (stretch)

**Success:** Player always knows what just happened and what happened 10 actions ago.

---

## Phase Completion Order
1 → 2 → 3

Phase 1 is data layer. Phase 2 is wiring. Phase 3 is UI.

---

## Definition of Done
- `ship.message` ref removed entirely
- Every game action generates a categorized log entry
- Event log visible in game client with scrollback
- No lost context between actions
