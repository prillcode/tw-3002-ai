# 12-01 — Rate Limiting & Action Budget

## Objective
Prevent automated scripts from gaining unfair speed advantages while preserving legitimate API usage.

## Context
- Worker runs on Cloudflare (edge, stateless)
- D1 is the only persistent store
- 42 endpoints across 10 categories (updated 2026-05-03): auth, galaxy, player, actions, crime, combat/stats, insurance, fighters, planets, mines, NPC
- No existing rate limiting
- `player_ships` table already has `last_action_at` column

## Design Decisions

### Rate Limits (per-IP, in-memory)
Use a simple in-memory Map with timestamps. Cloudflare Workers are short-lived but rate limit windows are short (1 min).

| Category | Endpoints | Limit | Key |
|----------|-----------|-------|-----|
| Auth | `POST /api/auth/register`, `POST /api/auth/verify` | 5/min | IP |
| Gameplay (POST) | `/api/player/ship`, `/api/player/ship/move`, `/api/player/pay-taxes`, `/api/player/commission`, `/api/action/trade`, `/api/action/combat`, `/api/action/rob`, `/api/action/steal`, `/api/action/upgrade`, `/api/insurance/buy`, `/api/news` (POST), `/api/fighters/buy`, `/api/fighters/deploy`, `/api/fighters/recall`, `/api/fighters/encounter/resolve`, `/api/planets/create`, `/api/planets/colonize`, `/api/planets/citadel/advance`, `/api/planets/qcannon`, `/api/planets/transport`, `/api/mines/buy`, `/api/mines/deploy`, `/api/mines/clear-limpets` | 10/min | playerId |
| Reads (auth) | `/api/player`, `/api/player/ship`, `/api/player/alignment`, `/api/player/stats`, `/api/insurance/status`, `/api/bounty/status`, `/api/notifications/digest`, `/api/port/crime-status`, `/api/fighters/sector`, `/api/planets/sector`, `/api/planets/citadel-costs`, `/api/planets/{id}`, `/api/mines/sector` | 60/min | playerId |
| Public reads | `/health`, `/api/galaxy`, `/api/galaxy/{id}`, `/api/galaxy/{id}/sectors`, `/api/galaxy/{id}/sector`, `/api/leaderboard`, `/api/news` (GET), `/api/bounty/board` | 60/min | IP |
| Admin/NPC | `POST /api/npc/tick`, `/api/npc/llm-health`, `/api/npc/model-benchmark` | 10/min | IP + secret |

### Action Budget (per-player, in D1)
- New column: `player_ships.action_points INTEGER DEFAULT 60`
- New column: `player_ships.action_points_refill_at DATETIME`
- Regeneration: 1 point per minute (60/hour, cap 60)
- Costs:
  - Move: 1
  - Trade: 1
  - Combat: 2
  - Rob / Encounter resolve: 2
  - Steal / Upgrade / Pay taxes / Commission: 1
  - Create ship: 1
  - Fighters buy/deploy/recall: 1 each
  - Planet create/colonize: 2 each
  - Planet citadel advance / qcannon / transport: 1 each
  - Mines buy/deploy/clear-limpets: 1 each
  - Insurance buy: 0 (it's a purchase, not an action)
- On insufficient points: return 403 `{ error: "Insufficient action points. Refills in X minutes." }`

## Tasks

### 1. Create `cloud/src/utils/rateLimit.ts`

```typescript
interface RateLimitEntry {
  count: number;
  windowStart: number;
}

const rateLimitMap = new Map<string, RateLimitEntry>();
const WINDOW_MS = 60_000; // 1 minute

export function checkRateLimit(key: string, maxRequests: number): { allowed: boolean; retryAfter?: number } {
  const now = Date.now();
  const entry = rateLimitMap.get(key);

  if (!entry || now - entry.windowStart > WINDOW_MS) {
    rateLimitMap.set(key, { count: 1, windowStart: now });
    return { allowed: true };
  }

  if (entry.count >= maxRequests) {
    const retryAfter = Math.ceil((entry.windowStart + WINDOW_MS - now) / 1000);
    return { allowed: false, retryAfter };
  }

  entry.count++;
  return { allowed: true };
}

export function getClientIP(request: Request): string {
  return request.headers.get('CF-Connecting-IP') || 'unknown';
}
```

### 2. Create `cloud/src/utils/actionBudget.ts`

```typescript
import type { D1Database } from '@cloudflare/workers-types';

const ACTION_COSTS: Record<string, number> = {
  move: 1,
  trade: 1,
  combat: 2,
  rob: 2,
  steal: 1,
  upgrade: 1,
  'create-ship': 1,
  'pay-taxes': 1,
  commission: 1,
  'insurance-buy': 0,
  'fighters-buy': 1,
  'fighters-deploy': 1,
  'fighters-recall': 1,
  'encounter-resolve': 2,
  'planet-create': 2,
  'planet-colonize': 2,
  'citadel-advance': 1,
  'qcannon': 1,
  'planet-transport': 1,
  'mines-buy': 1,
  'mines-deploy': 1,
  'clear-limpets': 1,
};

export async function checkAndDeductActionPoints(
  db: D1Database,
  playerId: number,
  galaxyId: number,
  action: string
): Promise<{ allowed: boolean; remaining: number; nextRefillMinutes: number }> {
  const cost = ACTION_COSTS[action] ?? 1;

  const ship = await db
    .prepare('SELECT action_points, action_points_refill_at FROM player_ships WHERE player_id = ? AND galaxy_id = ?')
    .bind(playerId, galaxyId)
    .first<{ action_points: number; action_points_refill_at: string | null }>();

  if (!ship) return { allowed: false, remaining: 0, nextRefillMinutes: 0 };

  // Calculate regenerated points
  const now = new Date();
  let points = ship.action_points;
  if (ship.action_points_refill_at) {
    const refillAt = new Date(ship.action_points_refill_at);
    const elapsedMinutes = Math.floor((now.getTime() - refillAt.getTime()) / 60_000);
    points = Math.min(60, points + elapsedMinutes);
  }

  if (points < cost) {
    const nextRefill = Math.ceil((cost - points));
    return { allowed: false, remaining: points, nextRefillMinutes: nextRefill };
  }

  const newPoints = points - cost;
  const newRefillAt = now.toISOString();

  await db
    .prepare('UPDATE player_ships SET action_points = ?, action_points_refill_at = ? WHERE player_id = ? AND galaxy_id = ?')
    .bind(newPoints, newRefillAt, playerId, galaxyId)
    .run();

  return { allowed: true, remaining: newPoints, nextRefillMinutes: 0 };
}
```

### 3. Modify `cloud/src/index.ts`

Wire rate limits into route blocks:
- **Public routes** (before auth): check `checkRateLimit('public:' + ip, 60)`
- **Auth routes**: check `checkRateLimit('auth:' + ip, 5)`
- **Auth-gated GETs** (reads): check `checkRateLimit('read:' + auth.playerId, 60)`
- **Auth-gated POSTs** (gameplay): check `checkRateLimit('gameplay:' + auth.playerId, 10)` + `checkAndDeductActionPoints`
- **Admin**: check `checkRateLimit('admin:' + ip, 10)`

Add `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset` headers to all responses.

Helper in cors.ts:
```typescript
export function jsonWithRateLimit(data: unknown, limit: number, remaining: number, resetAt: number, status = 200): Response {
  return new Response(JSON.stringify(data), {
    status,
    headers: {
      ...corsHeaders,
      'Content-Type': 'application/json',
      'X-RateLimit-Limit': String(limit),
      'X-RateLimit-Remaining': String(remaining),
      'X-RateLimit-Reset': String(resetAt),
    },
  });
}
```

## Notes
- In-memory rate limit map resets on Worker restart (acceptable for short windows)
- For stricter limits, consider Cloudflare Rate Limiting product (paid) or D1-backed counters
- Action budget is the primary anti-bot mechanism; rate limits are secondary
- `/api/bounty/board` moved from auth-reads to public reads (no auth actually required in code)
