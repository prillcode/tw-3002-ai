# 12-02 — API Documentation (Astro Content Collection)

## Objective
Create a complete public API documentation section on `playtradewars.net/api/` that teaches developers how to interact with the game API.

## Context
- Astro site uses content collections (`guide` already exists)
- `output: 'static'` — all pages pre-rendered at build time
- Terminal aesthetic throughout site
- API is public and unauthenticated for reads; bearer token for gameplay
- Base URL: `https://api.playtradewars.net`
- Current API version: `0.5.0`

## Design Decisions

### Content Collection: `api`
Schema: `title`, `description`, `section` ('overview' | 'reference' | 'tutorial'), `order`

### URL Structure
```
/api/                     → Overview + philosophy
/api/getting-started/     → First request, auth flow
/api/authentication/      → Register, verify, bearer tokens
/api/reference/           → Endpoint quick-reference table
/api/reference/galaxy/    → Galaxy read endpoints
/api/reference/player/    → Player/ship endpoints
/api/reference/actions/   → Trade, combat, upgrade
/api/reference/news/      → News, leaderboard, bounty
/api/tutorials/stats-dashboard/   → Read-only dashboard tutorial
/api/tutorials/news-bot/          → Discord webhook tutorial
```

### Terminal Aesthetic in Code Blocks
Use existing Shiki theme (`dracula`) for syntax highlighting. Wrap code blocks in a terminal-style container.

## Tasks

### 1. Update `web/main/src/content/config.ts`

Add `api` collection:
```typescript
const apiCollection = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    description: z.string().optional(),
    section: z.enum(['overview', 'reference', 'tutorial']).default('reference'),
    order: z.number().default(99),
  }),
});

export const collections = {
  guide: guideCollection,
  api: apiCollection,
};
```

### 3. Create `web/main/src/pages/api/[...slug].astro`

Same pattern as `guide/[...slug].astro`:
```astro
---
import { getCollection } from 'astro:content';
import ApiLayout from '../../layouts/ApiLayout.astro';

export async function getStaticPaths() {
  const docs = await getCollection('api');
  return docs.map((doc) => ({
    params: { slug: doc.slug },
    props: { doc },
  }));
}

const { doc } = Astro.props;
const { Content } = await doc.render();
---
<ApiLayout frontmatter={doc.data}>
  <Content />
</ApiLayout>
```

### 5. Write content files in `web/main/src/content/api/`

#### `introduction.md` (section: overview, order: 1)
- What this API is (game backend, REST, JSON)
- Base URL: `https://api.playtradewars.net`
- Version: `0.5.0`
- Philosophy: "The API is public because the game is played through it. Build tools, not bots."
- Rate limits and action budgets at a glance
- Link to Fair Play

#### `getting-started.md` (section: overview, order: 2)
- First request with curl (health check)
- Response format (always JSON, `{ data }` or `{ error }`)
- HTTP status codes used
- Rate limit headers explained

#### `authentication.md` (section: overview, order: 3)
- `POST /api/auth/register` — email only, no password
- `POST /api/auth/verify` — token returned directly
- Bearer token usage: `Authorization: Bearer <token>`
- Token expiry (7 days)
- No refresh token — just re-verify

#### `reference/galaxy.md` (section: reference, order: 10)
Endpoints:
- `GET /api/galaxy` — list galaxies
- `GET /api/galaxy/{id}` — galaxy details
- `GET /api/galaxy/{id}/sectors` — all sectors
- `GET /api/galaxy/{id}/sector?id={index}` — single sector

Each with: description, auth required?, request params, response shape, example curl.

#### `reference/player.md` (section: reference, order: 11)
Endpoints:
- `GET /api/player` — player profile
- `GET /api/player/ship?galaxyId={id}` — ship status
- `POST /api/player/ship` — create ship
- `POST /api/player/ship/move` — move to sector
- `GET /api/player/stats?galaxyId={id}` — kills/deaths/KDR

#### `reference/actions.md` (section: reference, order: 12)
Endpoints:
- `POST /api/action/trade` — buy/sell commodities
- `POST /api/action/combat` — fight NPC
- `POST /api/action/upgrade` — buy ship upgrade
- `POST /api/insurance/buy` — purchase insurance
- `GET /api/insurance/status?galaxyId={id}` — check insurance

#### `reference/news.md` (section: reference, order: 13)
Endpoints:
- `GET /api/news?galaxyId={id}&limit={n}` — galaxy news
- `GET /api/leaderboard?galaxyId={id}&limit={n}&sort={field}` — rankings
- `GET /api/bounty/board?galaxyId={id}` — wanted players
- `GET /api/bounty/status` — own wanted status
- `GET /api/notifications/digest?galaxyId={id}` — "while you were away"

#### `tutorials/stats-dashboard.md` (section: tutorial, order: 20)
Build a simple HTML dashboard that shows:
- Current ship stats (fetch `/api/player/ship`)
- Net worth over time (would need new endpoint or localStorage)
- Recent news (fetch `/api/news`)
- Pure frontend, no server needed
- Deploy to GitHub Pages or Vercel

#### `tutorials/news-bot.md` (section: tutorial, order: 21)
Build a Discord webhook that posts galaxy news:
- Poll `/api/news` every 5 minutes
- POST to Discord webhook URL
- Filter by type (combat, trade, event)
- Host on a free service (Glitch, Replit, Cloudflare Worker)
