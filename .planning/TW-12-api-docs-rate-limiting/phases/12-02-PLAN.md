# 12-02 — API Documentation (Astro Content Collection + Scalar UI)

## Objective
Create a complete public API documentation section on `playtradewars.net/api/` that teaches developers how to interact with the game API. Primary deliverable: **a Scalar API Reference UI** for interactive endpoint browsing. Supplementary: markdown prose docs for narrative content.

## Context
- Astro site uses content collections (`guide` already exists)
- `output: 'static'` — all pages pre-rendered at build time
- Terminal aesthetic throughout site
- API is public and unauthenticated for reads; bearer token for gameplay
- Base URL: `https://api.playtradewars.net`
- Current API version: `0.5.5`
- 42 endpoints across 10 categories (updated 2026-05-03)

## Design Decisions

### Primary: Scalar API Reference (`@scalar/api-reference`)
- **Scalar** is the modern replacement for Swagger UI — faster, prettier, better DX
- Define an **OpenAPI 3.1 spec** (`openapi.yaml` in the Astro project) as the single source of truth
- Render it on `/api/reference` using Scalar's Astro-compatible web component
- Spec lives in the codebase (versioned with the API itself), imported at build time
- Scalar provides: endpoint browser, try-it-out requests, auth helper, code samples, dark mode

### Supplementary: Markdown Content Collection
For narrative content that isn't endpoint reference (overview, tutorials, fair play), use the existing Astro content collection pattern.

### Content Collection: `api`
Schema: `title`, `description`, `section` ('overview' | 'tutorial'), `order`

### URL Structure
```
/api/                     → API overview + link to Scalar reference
/api/reference/           → Scalar UI (embedded @scalar/api-reference component)
/api/getting-started/     → First request, auth flow
/api/authentication/      → Register, verify, bearer tokens
/api/tutorials/stats-dashboard/   → Read-only dashboard tutorial
/api/tutorials/news-bot/          → Discord webhook tutorial
```

The Scalar UI replaces individual markdown reference pages (galaxy, player, actions, etc.). All endpoint details live in the OpenAPI spec.

### Why Scalar over manual markdown?
1. **Interactive** — users can send test requests from the docs
2. **Single source of truth** — the OpenAPI spec is machine-readable and can be validated
3. **Auto-generated request/response examples** — no more stale curl snippets
4. **Code sample generation** — curl, JS fetch, Python, etc. auto-generated
5. **Beautiful by default** — fits the terminal/dark theme aesthetic

## Tasks

### 1. Install Scalar
```bash
cd web/main && npm install @scalar/api-reference
```

### 2. Create `web/main/src/data/openapi.yaml`

Define the complete API spec. Covers all 42 endpoints:

**Public (no auth):**
- `GET /health`
- `POST /api/auth/register`, `POST /api/auth/verify`
- `GET /api/galaxy`, `GET /api/galaxy/{id}`, `GET /api/galaxy/{id}/sectors`, `GET /api/galaxy/{id}/sector`
- `GET /api/leaderboard`, `GET /api/news`, `GET /api/bounty/board`
- `GET/POST /api/npc/llm-health`, `GET/POST /api/npc/model-benchmark`

**Auth required — reads:**
- `GET /api/player`, `GET /api/player/ship`, `GET /api/player/alignment`, `GET /api/player/stats`
- `GET /api/insurance/status`, `GET /api/bounty/status`, `GET /api/notifications/digest`
- `GET /api/port/crime-status`, `GET /api/fighters/sector`, `GET /api/planets/sector`, `GET /api/planets/{id}`, `GET /api/planets/citadel-costs`, `GET /api/mines/sector`

**Auth required — gameplay:**
- `POST /api/player/ship`, `POST /api/player/ship/move`, `POST /api/player/pay-taxes`, `POST /api/player/commission`
- `POST /api/action/trade`, `POST /api/action/combat`, `POST /api/action/rob`, `POST /api/action/steal`, `POST /api/action/upgrade`
- `POST /api/insurance/buy`, `POST /api/news`
- `POST /api/fighters/buy`, `POST /api/fighters/deploy`, `POST /api/fighters/recall`, `POST /api/fighters/encounter/resolve`
- `POST /api/planets/create`, `POST /api/planets/colonize`, `POST /api/planets/citadel/advance`, `POST /api/planets/qcannon`, `POST /api/planets/transport`
- `POST /api/mines/buy`, `POST /api/mines/deploy`, `POST /api/mines/clear-limpets`

**Admin:**
- `POST /api/npc/tick`

### 3. Create `web/main/src/pages/api/reference.astro`

```astro
---
import Layout from '../../layouts/Layout.astro';
// Scalar web component is loaded client-side
---
<Layout title="API Reference — TW 3002 AI">
  <div class="scalar-container">
    <script
      id="api-reference"
      data-url="/api/openapi.yaml"
      type="module"
      src="https://cdn.jsdelivr.net/npm/@scalar/api-reference">
    </script>
  </div>
</Layout>
```

Or use the React/Vue component if preferred for tighter integration.

### 4. Update `web/main/src/content/config.ts`

Add `api` collection (for markdown prose pages only):
```typescript
const apiCollection = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    description: z.string().optional(),
    section: z.enum(['overview', 'tutorial']).default('overview'),
    order: z.number().default(99),
  }),
});

export const collections = {
  guide: guideCollection,
  api: apiCollection,
};
```
