# 12-03 — Fair Play & Terms of Service

## Objective
Set clear expectations about what is and isn't acceptable when using the public API, framing rate limits and action budgets as fairness mechanisms rather than obstacles.

## Context
- The API is public by necessity (web client uses it)
- The original TW2002 suffered from scripting/botting on BBSs
- We want to encourage educational use (dashboards, bots, tools) while preventing unfair gameplay automation
- The action budget system (60 actions/hour) makes perfect automation less impactful

## Design Decisions

### Tone
- Transparent, not adversarial
- "The API is public because that's how the game works. We trust you to play fair."
- Explain *why* limits exist (shared galaxy, fairness) rather than just listing rules

### What's Allowed
- Custom dashboards and stats trackers
- Discord/news notification bots
- Data analysis and visualization
- Learning API programming via the game
- Alt accounts for testing (within reason)

### What's Not Allowed
- Automated trading scripts (buy low / sell high without human input)
- Automated combat bots (auto-attack NPCs or players)
- Automated movement scripts (optimal route farming)
- Coordinate sharing bots (real-time sector data feeds to give advantage)
- Multiple accounts working together to manipulate markets
- Any script that makes decisions for the player

### Enforcement
- Rate limits catch speed-based abuse
- Action budgets catch volume-based abuse
- No IP bans for first offenses — throttle only
- Persistent botting may result in account restrictions

## Tasks

### 1. Create `web/main/src/content/api/fair-play.md`

Content structure:
```markdown
---
title: Fair Play
description: Rules and philosophy for using the TW 3002 AI API
section: overview
order: 4
---

# Fair Play

## The API Is Public On Purpose

The TW 3002 AI API is public because the web client at portal.playtradewars.net uses it for every action. You can see every request in your browser's DevTools. We don't hide the API — we document it.

## Why Limits Exist

The galaxy is shared. If one player runs a script that makes 1,000 trades per minute, everyone else's experience degrades. Rate limits and action budgets aren't anti-developer — they're pro-fairness.

## Action Budget: The Great Equalizer

Every player gets 60 action points per hour. Moving, trading, fighting — they all cost points. A script can't do *more* than a human. It can only optimize what's available.

## What's Encouraged

✅ **Stats Dashboards** — Track your own progress. Build a prettier HUD than ours.

✅ **News Bots** — Pipe galaxy news to your Discord server. Keep your corp informed.

✅ **Data Projects** — Analyze trade routes, price histories, NPC patterns. Share your findings.

✅ **Learning** — This API is a real production backend. Use it to learn REST, auth, rate limiting, and game state management.

## What's Not Allowed

❌ **Auto-Traders** — Scripts that buy and sell without you clicking a button.

❌ **Auto-Combat** — Scripts that attack NPCs or players automatically.

❌ **Route Optimizers with Execution** — Calculating the best route is fine. Flying it for you is not.

❌ **Real-Time Feeds** — Broadcasting live sector data to give players an unfair map advantage.

## Enforcement

- **Soft limits:** Rate throttling (429 responses)
- **Action limits:** 60 points/hour max
- **Hard limits:** Persistent botting may result in account suspension

## We Trust You

TradeWars was always a game of strategy, not reflexes. A bot can't negotiate a truce, plan a blockade, or bluff in PvP. Build cool tools. Don't build tools that play the game for you.
```
