# 13-04 — Mines (Limpet + Armid)

## Objective
Add mine warfare as the second defensive layer after fighters: Limpet mines attach and impair, Armid mines detonate and damage on sector entry.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- Depends on Phase 13-02 encounter gating (sector entry is now interceptable)
- Should follow Phase 13-03 sequencing work so mine steps can be inserted cleanly in operation order
- No corporation mine-sharing; ownership is individual only

## Scope
### In-scope
- Mine purchase at StarDock
- Mine deployment in non-FedSpace sectors
- Limpet attachment checks on entry
- Armid detonation checks on entry
- Mine visibility rules (owner-visible, hostile hidden)
- Mine removal at StarDock (limpets)

### Out-of-scope
- Advanced scanner minigame
- Photon-clearing interactions (Phase 13-05+)
- Corp mine pools

## Data Model

### New table: `sector_mines`
```sql
CREATE TABLE sector_mines (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  galaxy_id INTEGER NOT NULL,
  sector_index INTEGER NOT NULL,
  owner_id INTEGER NOT NULL,
  limpet_count INTEGER NOT NULL DEFAULT 0,
  armid_count INTEGER NOT NULL DEFAULT 0,
  deployed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(galaxy_id, sector_index, owner_id)
);
```

### `player_ships` additions
- `limpets` integer default 0 (inventory on ship)
- `armids` integer default 0 (inventory on ship)
- `limpet_attached` integer default 0 (how many currently attached to ship)

## API Endpoints

### Purchase / deploy
- `POST /api/mines/buy`
  - body: `{ galaxyId, type: 'limpet' | 'armid', quantity }`
  - StarDock only
- `POST /api/mines/deploy`
  - body: `{ galaxyId, sectorId, type, quantity }`
  - must be in sector, non-FedSpace

### Info
- `GET /api/mines/sector?galaxyId=&sectorId=`
  - owner gets full detail
  - non-owner gets obscured/high-level signals only

### Removal
- `POST /api/mines/clear-limpets`
  - StarDock only
  - fee: 5,000 cr per limpet attached

## Entry Resolution Rules

### Limpet (on sector entry)
- Triggered before armid and before fighter engagement
- Each hostile limpet has chance to attach (tunable constant)
- Attached limpets increase `player_ships.limpet_attached`
- Initial gameplay effect (v1):
  - adds operation-log warning
  - optional turn tax (e.g. +1 turn every N moves) deferred toggle

### Armid (on sector entry)
- Triggered after limpet step
- Each hostile armid has 50% detonation chance
- On detonation:
  - armid consumed from sector mine pool
  - ship takes hull damage (tunable per mine)
- If hull reaches 0, route through `resolveDefeat()` with cause `fighter` (or new `mine` cause if added)
