## Scope
### In-scope
- Blockade detection and severity signaling
- Q-cannon entry damage (sector + atmospheric hook integration)

### Out-of-scope
- Corporations/team blockades (explicitly excluded)
- Photon torpedo suppression logic (future)
- Terra Hazz/nav-hazard special blockades

## Blockade Model

### Detection metric
For each sector, compute blockade score from:
- hostile deployed fighters
- hostile mines (if available)
- recent entry defeats (rolling window)

Suggested tiers:
- **Light pressure:** score >= T1
- **Active blockade:** score >= T2
- **Fortress blockade:** score >= T3

### Behavior
- Add blockade flag to sector info responses
- UI shows warning badge in warp lanes and sector header
- No hard lockout; blockade increases risk, not impossible access

## Q-Cannon Integration

### Required TW-14 inputs
- planet ownership
- citadel level
- sector cannon percentage (`sect_cannon_pct`)
- atmospheric cannon percentage (`atmo_cannon_pct`)
- ore pool / production backing values

### Entry sequence placement
1. Nav hazard
2. Limpets
3. Armids
4. **Q-cannon fire**
5. Fighters
6. Ship-to-ship

### Damage formulas (as planned)
- Sector cannon: `(TotalOre * SectPct) / 3`
- Atmospheric cannon: `TotalOre * AtmoPct * 2`

Use tunable clamps to prevent one-shot frustration in first pass.

## Loot Rebalance

### Trigger condition
Only raise loot after all defensive layers are active:
- fighters
- mine layer
- q-cannon layer (or equivalent planetary defense)

### Change
- In `resolveDefeat()` constants:
  - `LOOT_CREDITS_PCT` from `0.25` → `0.50`
- Keep insurance mitigation in place to reduce wipe severity for prepared players

### Safety checks
- monitor net-worth churn and rage-quit signals
- rollback knob via env/config constant if economy destabilizes
