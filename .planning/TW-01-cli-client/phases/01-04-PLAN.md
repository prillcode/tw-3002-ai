# PLAN: TW-01 Phase 4 — Sector View

**Work Item:** TW-01-cli-client  
**Phase:** 4 of 7  
**Estimated Duration:** 3-4 hours  
**Goal:** Main galaxy navigation screen with ASCII sector map, ship status, and connected sectors

---

## Objective
Build the primary gameplay screen: the sector view. This is where players spend most of their time — navigating the galaxy, viewing their ship status, and deciding where to jump next. Must evoke the classic Trade Wars "sector command" feel.

---

## Context

### What We Have (Phase 3)
- Welcome flow complete (title → menu → ship naming)
- Ship name stored and displayed
- Placeholder SectorScreen with static text
- Player enters sector 42 (FedSpace) after naming ship

### What We're Building (Phase 4)
- Visual ASCII sector map (center display)
- Interactive connected sectors list (select and jump)
- Real-time ship status panel (credits, cargo, hull, fuel)
- Sector information (port type, danger, notes)
- Smooth keyboard navigation between sectors

---

### Task 1: Create SectorMap Component (60 min)
**Create `src/components/SectorMap.tsx`**

**Requirements:**
- ASCII visualization of current sector + neighbors
- Center: current sector (highlighted)
- Surrounding: connected sectors (selectable)
- Visual connections (lines/dots between sectors)
- Port indicators (P for port, empty for none)

**Visual Target:**
```
         [ 41 ]←P
            ↑
    [ 40 ]←[ 42 *]→[ 43 ]←P
            ↓
         [ 52 ]
```
(* marks current sector, P marks port, arrows show connections)

**Alternative simpler version:**
```
      ┌─────────┐
      │ SECTOR  │
      │   42  ★ │  ← current
      └────┬────┘
           │
   ┌───────┼───────┐
   │       │       │
[ 41←P] [ 42★ ] [ 43←P]
   │               │
[ 52 ]          [ 44 ]
```

**API:**
```tsx
<SectorMap
  currentSector={42}
  neighbors={[41, 43, 52]}
  ports={{41: 'II', 43: 'I'}}
  selectedIndex={0}
/>
```

**Implementation notes:**
- Use Box + Text components for positioning
- Fixed-width layout (assume 80 char terminal)
- Highlight selected neighbor with different color
- Current sector gets special marker (★ or * or bold)

### Task 7: Wire Up Sector State (30 min)
**Update `App.tsx` and `SectorScreen`**

**Requirements:**
- Track current sector in App state
- Pass sector data to SectorScreen
- Handle "jump" to new sector (update current)
- Update ship status (fuel/turns decrease on jump)

**State changes:**
```tsx
// In App.tsx
const [currentSector, setCurrentSector] = useState(42);
const [shipStatus, setShipStatus] = useState({
  credits: 5000,
  cargo: {ore: 0, organics: 0, equipment: 0, total: 0, max: 100},
  hull: 100,
  turns: 100
});
```

**Handle jump:**
```tsx
const handleJump = (sectorId: number) => {
  setCurrentSector(sectorId);
  setShipStatus(prev => ({
    ...prev,
    turns: prev.turns - 1 // each jump costs 1 turn
  }));
};
```

**Verification:**
- [ ] Current sector tracks properly
- [ ] Jumping updates sector
- [ ] Turns decrement
- [ ] ShipStatus receives updates
