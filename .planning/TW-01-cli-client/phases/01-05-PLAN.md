# PLAN: TW-01 Phase 5 — Market Interface

**Work Item:** TW-01-cli-client  
**Phase:** 5 of 7  
**Estimated Duration:** 3-4 hours  
**Goal:** Trading screen with buy/sell functionality, commodity prices, and transaction confirmation

---

## Objective
Build the trading interface where players buy and sell commodities at ports. This is the core economic loop of the game — the interface must clearly show prices, cargo capacity, profit potential, and execute trades safely.

---

## Context

### What We Have (Phase 4)
- Sector navigation working with 10-sector galaxy
- Ports have classes (I, II, III) and commodity types
- Ship status displays (credits, cargo capacity)
- Pressing 'M' from sector screen goes to market
- MarketScreen is a placeholder with static prices

### What We're Building (Phase 5)
- Dynamic market prices based on port type
- Buy and sell interface with quantity selection
- Cargo capacity enforcement (can't exceed hold limit)
- Credit validation (can't spend more than you have)
- Transaction confirmation and feedback
- Port inventory limits (can't buy more than port has)
- Visual price indicators (good deals highlighted)

## Economy Reference

**Base prices:**
- Ore: ~100 credits
- Organics: ~50 credits
- Equipment: ~200 credits

**Port class modifiers:**
- Class I: ±5% (best prices)
- Class II: ±10%
- Class III: ±15% (worst prices)

**Buy/Sell spread:**
- Ports buy for less than they sell (profit margin)
- Example: Port buys Ore at 95, sells at 120 (25% margin)

**Cargo space:**
- 1 unit = 1 ton
- Merchant ship: 100 units
- Upgradable in later phases
