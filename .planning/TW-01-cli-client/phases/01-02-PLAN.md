# PLAN: TW-01 Phase 2 — Core TUI Components

**Work Item:** TW-01-cli-client  
**Phase:** 2 of 7  
**Estimated Duration:** 4-6 hours  
**Goal:** Reusable component library for game screens

---

## Objective
Build the foundational TUI components that all game screens will use: Box containers, colored Text, keyboard-navigable Menus, persistent StatusBar, and React hooks for screen routing and input handling. These components will be composed to build Welcome, Sector, Market, and Combat screens.

---

## Context

### What We Have (Phase 1)
- Bun + Ink project running
- Single static screen with Box and Text
- No interactivity yet

### What We're Building (Phase 2)
- Component library in `src/components/`
- Custom hooks in `src/hooks/`
- Screen router for multi-screen navigation
- Keyboard input handling

### Component Architecture
```
src/
├── components/
│   ├── Box.tsx      # Bordered container with styling variants
│   ├── Text.tsx     # Colored text (green=ok, red=danger, yellow=warn)
│   ├── Menu.tsx     # Keyboard-navigable list with selection
│   └── StatusBar.tsx # Persistent bottom bar showing keys/help
├── hooks/
│   ├── useKeyHandler.ts  # Centralized keyboard input
│   └── useScreen.ts      # Screen router (welcome/sector/market/combat)
└── screens/ (created in Phase 3, but router needs to reference)
    ├── WelcomeScreen.tsx
    ├── SectorScreen.tsx
    └── MarketScreen.tsx
```

### Task 6: useScreen Hook (60 min)
**Create `src/hooks/useScreen.ts` — screen router for multi-screen navigation.**

**Requirements:**
- Track current screen: 'welcome' | 'sector' | 'market' | 'combat' | 'settings'
- Provide `navigateTo(screen)` function
- Provide `goBack()` to previous screen (optional stack)
- Screen history stack (for back navigation)

**API:**
```tsx
const { currentScreen, navigateTo, goBack, canGoBack } = useScreen({
  initial: 'welcome'
});

// Use in component:
if (currentScreen === 'welcome') return <WelcomeScreen onStart={() => navigateTo('sector')} />;
if (currentScreen === 'sector') return <SectorScreen onTrade={() => navigateTo('market')} />;
```

**State shape:**
```typescript
interface ScreenState {
  current: ScreenName;
  history: ScreenName[];  // Stack for goBack
}
type ScreenName = 'welcome' | 'sector' | 'market' | 'combat' | 'settings';
```

**Implementation notes:**
- Use React useState for screen stack
- Push to history on navigate (except for goBack itself)
- Limit history size (10?) to prevent memory growth

**Verification:**
- [ ] Can navigate forward to any screen
- [ ] goBack returns to previous screen
- [ ] canGoBack reflects history state
- [ ] Screen component re-renders on change
