# 01-07-PLAN — Polish & Packaging

## Objective
Prepare the CLI for distribution and real users. Fix remaining UI inconsistencies, add helpful error messages, set up npm packaging, and document everything. Make `tw3002` installable via `npm install -g tw3002` and runnable without a git clone.

## Context
- TW-01 Phases 1-6b are complete — the CLI is feature-complete for local single-player
- TW-02 integration may add more screens (combat, etc.) — this plan assumes those exist
- Current install: `git clone` + `bun install` + `bun run dev` — not user-friendly
- No npm package exists yet
- README is minimal

## Approach

Packaging work happens in two layers:
1. **Code quality** — error handling, edge cases, help text
2. **Distribution** — npm package, binary releases, documentation

## Tasks

### 1. Error handling & graceful exits

**Current gaps:**
- `process.exit(0)` on quit — abrupt, no cleanup
- No handling for terminal resize during gameplay
- No handling for SIGINT (Ctrl+C) during input
- Database errors can crash the app

**Add to `cli/src/index.tsx` or a new `cli/src/utils/errors.ts`:**
- [ ] `useExitHandler()` hook — intercepts Ctrl+C, saves game before exit
- [ ] Terminal resize listener — clear + re-render on resize
- [ ] DB error wrapper — all DB calls wrapped in try/catch with user-friendly message
- [ ] `panic()` function — shows red error box + save attempt + exit

### 5. Package for npm

**Update root and CLI `package.json`:**

```json
// cli/package.json
{
  "name": "tw3002",
  "version": "0.1.0",
  "description": "Terminal-native space trading game with AI-driven NPCs",
  "bin": {
    "tw3002": "./tw3002"
  },
  "files": ["tw3002", "README.md"],
  "engines": {
    "bun": ">=1.0.0"
  },
  "scripts": {
    "dev": "bun run src/index.tsx",
    "build": "bun build src/index.tsx --outfile=tw3002 --compile",
    "typecheck": "tsc --noEmit",
    "prepublishOnly": "bun run build"
  }
}
```

- [ ] Add `bin` field pointing to compiled binary
- [ ] Add `files` whitelist for npm publish
- [ ] Add `keywords`: `["game", "terminal", "tui", "space", "trading", "bbs"]`
- [ ] Add `license` field (MIT recommended)
- [ ] Add `repository` field
- [ ] Ensure `tw3002` binary is executable (`chmod +x`)

**Note:** Compiled Bun binaries are platform-specific. For cross-platform npm distribution, consider:
- Option A: Publish platform-specific packages (`tw3002-linux`, `tw3002-macos`, `tw3002-windows`) with a wrapper package
- Option B: Source distribution — npm installs source, `postinstall` compiles binary
- Option C: GitHub Releases with pre-built binaries, npm package just downloads the right one

**Recommended for MVP:** Option C — GitHub Releases for binaries, npm package as optional convenience.

### 8. Binary release workflow (GitHub Actions)

**`.github/workflows/release.yml`:**
```yaml
name: Release
on:
  push:
    tags: ['v*']
jobs:
  build:
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v1
      - run: bun install
      - run: cd cli && bun run build
      - uses: softprops/action-gh-release@v1
        with:
          files: cli/tw3002
```

## Notes
- **Binary size:** Bun compiles to ~100MB. This is normal (bundles runtime). Document it.
- **Platform limits:** One binary per OS. GitHub Releases is the pragmatic approach.
- **npm registry:** Don't publish to npm until you're ready for users. Set up the config, test with `npm pack`, but hold off on `npm publish` until after alpha.
- **This phase can run in parallel with TW-02 Phase 8** (integration & test). TW-02 tests the game, TW-01 Phase 7 packages it.
