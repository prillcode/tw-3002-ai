# PLAN: TW-01 Phase 6 — Local Persistence (SQLite)

**Work Item:** TW-01-cli-client  
**Phase:** 6 of 7  
**Estimated Duration:** 2-3 hours  
**Goal:** Save and load game state using SQLite (bun:sqlite)

---

## Objective
Enable players to save their progress and resume later. Implement a SQLite database using Bun's built-in `bun:sqlite` module to persist ship state, current sector, cargo, and credits between game sessions.

---

## Context

### What We Have (Phase 5)
- Complete gameplay loop: navigate, trade, jump between sectors
- All state lives in React useState (ephemeral)
- Game resets to defaults on every launch
- No way to resume a previous session

### What We're Building (Phase 6)
- SQLite database schema for game state
- Auto-save on quit or significant events
- Load game from main menu "Continue" option
- Multiple save slots (optional, could be single save for MVP)
- Database file stored in user's home directory

---

## Tasks

### Task 1: Set Up SQLite Database (30 min)
**Create `src/db/database.ts`**

**Requirements:**
- Initialize SQLite using `bun:sqlite`
- Create database file in `~/.tw3002/saves.db` (or platform-appropriate location)
- Create tables: `games`, `cargo`, `sectors_visited` (optional)

**Schema:**
```sql
CREATE TABLE IF NOT EXISTS saves (
  id INTEGER PRIMARY KEY,
  ship_name TEXT NOT NULL,
  credits INTEGER DEFAULT 5000,
  current_sector INTEGER DEFAULT 42,
  cargo_ore INTEGER DEFAULT 0,
  cargo_organics INTEGER DEFAULT 0,
  cargo_equipment INTEGER DEFAULT 0,
  hull INTEGER DEFAULT 100,
  turns INTEGER DEFAULT 100,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**Implementation:**
```typescript
import { Database } from 'bun:sqlite';

const DB_PATH = `${process.env.HOME}/.tw3002/saves.db`;

export const initDatabase = () => {
  const db = new Database(DB_PATH);
  db.run(`CREATE TABLE IF NOT EXISTS saves (...)`);
  return db;
};
```

**Verification:**
- [ ] Database file created on first run
- [ ] Schema initializes correctly
- [ ] Can query and insert data

### Task 2: Create Save/Load Functions (30 min)
**Create `src/db/saveGame.ts` and `src/db/loadGame.ts`**

**Save function:**
```typescript
export const saveGame = (db: Database, gameState: GameState) => {
  db.run(`
    INSERT INTO saves (ship_name, credits, current_sector, ...)
    VALUES (?, ?, ?, ...)
    ON CONFLICT(id) DO UPDATE SET ...
  `, [...values]);
};
```

**Load function:**
```typescript
export const loadGame = (db: Database): GameState | null => {
  const row = db.query("SELECT * FROM saves ORDER BY updated_at DESC LIMIT 1").get();
  if (!row) return null;
  return {
    shipName: row.ship_name,
    credits: row.credits,
    ...
  };
};
```

**List saves:**
```typescript
export const listSaves = (db: Database): SaveSummary[] => {
  return db.query("SELECT id, ship_name, updated_at FROM saves").all();
};
```

**Verification:**
- [ ] Save writes data to SQLite
- [ ] Load retrieves correct data
- [ ] Updates modify existing row (not create duplicates)

### Task 3: Wire Up Auto-Save (30 min)
**Integrate save/load into App.tsx**

**Auto-save triggers:**
- On quit (Q key) — save current state
- After major events (trade completed, sector jumped)
- Optional: periodic auto-save every N actions

**Implementation in App.tsx:**
```typescript
const db = useMemo(() => initDatabase(), []);

// Save on state change
useEffect(() => {
  saveGame(db, { shipState, currentSectorId });
}, [shipState, currentSectorId]);

// Or explicit save function
const handleQuit = () => {
  saveGame(db, gameState);
  process.exit(0);
};
```

**Verification:**
- [ ] Game saves on quit
- [ ] Database file grows with save data
- [ ] Can inspect with `sqlite3` CLI

### Task 5: Handle Save Corruption/Errors (20 min)
**Graceful error handling**

**Requirements:**
- Try/catch around all DB operations
- Fallback to defaults if load fails
- Message to user if save is corrupted
- Option to start new game if load fails

**Implementation:**
```typescript
export const safeLoadGame = (db: Database): GameState | null => {
  try {
    return loadGame(db);
  } catch (err) {
    console.error('Failed to load save:', err);
    return null;
  }
};
```

**Verification:**
- [ ] Corrupted save doesn't crash game
- [ ] Error message displayed to user
- [ ] Can start new game after failed load
