# PLAN: TW-01 Phase 1 — Project Scaffold

**Work Item:** TW-01-cli-client  
**Phase:** 1 of 7  
**Estimated Duration:** 2-3 hours  
**Goal:** Bun + Ink project running with hello-world TUI

---

## Objective
Create the CLI project foundation: Bun runtime configured, Ink (React for terminals) installed, TypeScript working, and a basic "Hello TW 3002" TUI rendering. This establishes the development environment and build pipeline.

---

## Context

### Stack Decisions (from PRD)
- **Runtime:** Bun (TypeScript-native, fast, built-in SQLite later)
- **TUI Framework:** Ink (React for terminals, component-based)
- **Build Target:** Single executable binary

### What We're Building
A terminal-native CLI that renders React components using Ink. Think of it as a web app that renders to the terminal instead of a browser.

### Directory Structure Target
```
cli/
├── package.json
├── tsconfig.json
├── src/
│   └── index.tsx    # Entry point
├── build/           # Compiled output
└── README.md
```

---

## Tasks

### Task 1: Initialize Bun Project (15 min)
**Create the `cli/` directory and initialize Bun.**

```bash
mkdir -p tw-3002-ai/cli
cd tw-3002-ai/cli
bun init
```

**Verification:**
- [ ] `package.json` exists with `"name": "tw3002"`
- [ ] `bun --version` shows version (1.0+)
- [ ] `index.ts` created by init

---

### Task 2: Install Dependencies (15 min)
**Add Ink and React types.**

```bash
bun add ink react
bun add -d @types/react @types/bun
```

**Packages to install:**
- `ink` — React for terminals
- `react` — Peer dependency for Ink
- `@types/react` — TypeScript definitions
- `@types/bun` — Bun runtime types

**Verification:**
- [ ] `node_modules/ink/` exists
- [ ] `package.json` lists all dependencies
- [ ] No peer dependency warnings

---

### Task 3: Configure TypeScript (20 min)
**Set up `tsconfig.json` for Bun + React.**

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "outDir": "./build",
    "rootDir": "./src"
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "build"]
}
```

**Key settings:**
- `jsx: "react-jsx"` — Required for Ink components
- `module: "ESNext"` — Bun uses ES modules
- `outDir: "./build"` — Compiled output location

**Verification:**
- [ ] `tsconfig.json` valid JSON
- [ ] `tsc --noEmit` passes (or skip if no global tsc)

---

### Task 4: Create Entry Point (30 min)
**Write `src/index.tsx` with basic Ink app.**

```tsx
import React from 'react';
import { render, Text, Box } from 'ink';

const App = () => {
  return (
    <Box borderStyle="round" padding={1}>
      <Text color="green">Hello TW 3002 AI</Text>
    </Box>
  );
};

render(<App />);
```

**Explanation:**
- `Box` — Container with optional border/padding
- `Text` — Text with color support
- `render()` — Mounts React tree to terminal

**Verification:**
- [ ] File at `src/index.tsx`
- [ ] TypeScript compiles without errors

---

### Task 5: Configure Scripts (15 min)
**Add dev and build scripts to `package.json`.**

```json
{
  "name": "tw3002",
  "version": "0.1.0",
  "type": "module",
  "scripts": {
    "dev": "bun run src/index.tsx",
    "build": "bun build src/index.tsx --outfile=tw3002 --compile",
    "typecheck": "tsc --noEmit"
  },
  "dependencies": {
    "ink": "^4.x",
    "react": "^18.x"
  },
  "devDependencies": {
    "@types/react": "^18.x",
    "@types/bun": "latest"
  }
}
```

**Scripts explained:**
- `dev` — Run directly with Bun (fast, no compile)
- `build` — Compile to standalone `tw3002` binary
- `typecheck` — Verify TypeScript without emitting

**Verification:**
- [ ] `package.json` is valid JSON
- [ ] Can parse with `cat package.json | jq .`
