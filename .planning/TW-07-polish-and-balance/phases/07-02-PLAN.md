# 07-02 — Ship Stats from Class + Upgrades

## Objective
Remove hardcoded ship stats from CloudSectorScreen and compute them dynamically from ship class and upgrades.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- Currently: `maxCargo: 120`, `maxHull: s.hull` — all ships feel identical
- Engine has `computeEffectiveStats(classId, upgrades)` that returns correct values
- Cloud DB stores `class_id` and `upgrades_json`

## Tasks
1. In `loadShip()`, parse `upgrades_json` from the API response
2. Call `computeEffectiveStats(s.class_id, upgrades)` to get effective stats
3. Set `maxCargo`, `maxHull`, `maxShield`, `maxTurns` from computed values
4. Remove all hardcoded defaults from CloudSectorScreen
5. Update `handleCreateShip()` to set correct base stats per class

## Verification
- [ ] Merchant: 120 cargo, 100 hull, 80 turns
- [ ] Scout: 60 cargo, 80 hull, 120 turns
- [ ] Interceptor: 70 cargo, 120 hull, 80 turns
- [ ] Upgrades increase stats correctly (Cargo Holds +20, etc.)

## Done When
- No hardcoded stats in CloudSectorScreen
- All three classes feel distinct
- Upgrades apply correctly

## Notes
- `computeEffectiveStats` is already exported from `@tw3002/engine`
- The cloud DB stores the raw hull/shield values, not effective maxima
- Consider adding `max_hull`, `max_shield` columns to `player_ships` for caching
