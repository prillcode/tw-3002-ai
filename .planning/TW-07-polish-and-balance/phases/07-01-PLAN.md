# 07-01 — Port Inventory Refresh

## Objective
Ensure port supply is shared across all players by refreshing inventory data on every sector visit.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- Currently, CloudSectorScreen fetches the galaxy once and never refreshes sector data
- `cloudGetSector()` already returns `port_inventory_json`

## Tasks
1. In `CloudSectorScreen`, call `loadSector()` (which calls `cloudGetSector()`) on every successful jump
2. Store the parsed inventory on the sector object
3. Trade overlay reads from this refreshed inventory
4. Consider adding `GET /api/galaxy/:id/sector/:id/inventory` for lighter polling
5. Display current supply alongside prices in trade overlay

## Verification
- [ ] Player A buys 50 ore. Supply drops from 1000 to 950.
- [ ] Player B enters same sector. Trade overlay shows supply 950.
- [ ] `curl` confirms D1 inventory updated after trade.

## Done When
- Port inventory updates are visible to all players
- No stale data in trade overlay
- Supply never goes negative

## Notes
- Deducting supply happens in `POST /api/action/trade` — already implemented
- The client just needs to refresh its view after each trade or sector entry
- For the web client, this means polling sector data every 30s when docked
