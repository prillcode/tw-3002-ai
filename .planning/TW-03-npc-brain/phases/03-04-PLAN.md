# 03-04-PLAN — NPC Memory & Reputation

## Objective
Make NPC memory structures actually get populated. When you fight a pirate and win, he remembers. When you trade with a merchant at good prices, she notes it. Memory creates emergent narratives: grudges, alliances, market knowledge. Add player reputation tracking so NPCs react to you based on past interactions.

## Context
- TW-03 Phase 1 defined memory types: `lastActions`, `grudges`, `alliances`, `marketObservations`
- But only `lastActions` is populated (by `brain.ts` after each action)
- `grudges`, `alliances`, `marketObservations` are always empty arrays
- No player reputation system exists — NPCs don't know if you're a friend or foe
- This is what makes NPCs feel *personal* rather than interchangeable

## Approach

Hook memory writes into existing game events:

```
Player wins combat vs NPC "Void Fang"
  → NPC adds grudge: {target: player, reason: "Destroyed my ship", severity: 8}
  → NPC remembers: last action = "was defeated by player"

Player sells ore at good price to Trader "Kira"
  → NPC adds alliance: {target: player, reason: "Fair trader"}
  → NPC notes market observation: {sector: 42, commodity: ore, price: 95}

Player attacks Trader "Kira" (who had alliance)
  → Alliance broken
  → Grudge added: {target: player, reason: "Betrayed alliance", severity: 9}
```

Memory affects future decisions through the prompt — LLM (or rule-based logic) sees grudges/alliances and acts accordingly.

## Tasks

### 1. Add memory mutation helpers (`packages/engine/src/npcs/memory.ts`)

```typescript
export function addGrudge(npc: NPC, targetId: string, targetName: string, reason: string, severity: number): NPC;
export function removeGrudge(npc: NPC, targetId: string): NPC;
export function addAlliance(npc: NPC, targetId: string, targetName: string): NPC;
export function breakAlliance(npc: NPC, targetId: string): NPC;
export function addMarketObservation(npc: NPC, sectorId: number, commodity: Commodity, price: number): NPC;
export function decayMemory(npc: NPC): NPC; // reduce severity, remove old observations
```

**Grudge rules:**
- Severity 1-10
- Combat loss → severity = 5 + (damage taken / 10), capped at 10
- Betrayal (attacking an ally) → severity = 9
- Max 5 grudges per NPC (drop lowest when adding a 6th)

**Alliance rules:**
- Max 3 alliances per NPC
- Formed after 3+ profitable trades or 1 rescue
- Broken by any attack
- NPCs with alliance won't attack each other

**Market observation rules:**
- Max 10 observations per NPC
- Oldest dropped when adding 11th
- Observations used by traders to find profitable routes

**Decay rules (called once per tick):**
- Grudge severity -1 per 5 turns
- Grudges below severity 2 removed
- Market observations older than 20 turns removed

### 2. Hook memory writes into combat

In `cli/src/index.tsx` `handleCombatEnd()`:
```typescript
// After combat, update NPC memory
if (combatEnemy came from an NPC) {
  const npc = findNPCByName(npcs, combatEnemy.name);
  if (npc) {
    if (result.victory) {
      // Player won → NPC holds grudge
      const updated = addGrudge(npc, 'player', playerName, 'Defeated me in combat', 7);
      updateNPC(updated);
    } else if (result.fled) {
      // Player fled → NPC feels confident
      const updated = addGrudge(npc, 'player', playerName, 'Ran from our fight', 3);
      updateNPC(updated);
    }
  }
}
```

**Problem:** The current combat system doesn't track which NPC the enemy came from. Need to pass the NPC ID through combat.

**Fix:** Add `npcId?: string` to the CombatScreen/Combatant flow.

### 3. Hook memory writes into trading

In `cli/src/screens/MarketScreen.tsx` (or via `executeTrade` in engine):
After a successful trade, if there are NPC traders in the same sector:
- They observe the transaction
- If player sold at good prices → potential alliance
- If player bought all stock → mild grudge ("cornered the market")

For MVP simplicity: just add a market observation for all NPC traders in the sector.
```typescript
// After player trade:
const traders = npcs.filter(n => n.currentSectorId === currentSectorId && n.persona.type === 'trader');
for (const trader of traders) {
  const updated = addMarketObservation(trader, currentSectorId, commodity, price);
  updateNPC(updated);
}
```

### 4. Add reputation tracker (`packages/engine/src/npcs/reputation.ts`)

```typescript
export interface Reputation {
  targetId: string;
  targetName: string;
  score: number;     // -100 (hated) to +100 (loved), 0 = neutral
  interactions: number;
  lastInteraction: string;
}

export function updateReputation(
  npc: NPC,
  targetId: string,
  targetName: string,
  delta: number,     // +5 for good trade, -10 for combat loss, etc.
): NPC;
```

Store reputation inside `NPCMemory`:
```typescript
interface NPCMemory {
  lastActions: NPCActionRecord[];
  grudges: Grudge[];
  alliances: Alliance[];
  marketObservations: MarketObservation[];
  reputation: Record<string, Reputation>;  // targetId → reputation
}
```

**Reputation effects on behavior:**
- Score > +30: NPC won't attack, offers better trade prices
- Score < -30: NPC always attacks on sight, calls allies
- Score < -60: NPC actively hunts player (moves toward player's sector)

### 5. Update prompts to include memory

In `packages/engine/src/llm/prompts.ts` `buildPrompt()`:

```
{grudgeList}

```

This gives the LLM (and rule-based logic) context to make personalized decisions.

### 6. Update rule-based brain to use reputation

In `brain.ts`:
- Raider checks `reputation['player'].score` before attacking
  - Score < -50: always attack
  - Score > +20: don't attack (unless aggression > 0.9)
- Trader checks `reputation['player'].score` for price adjustments
  - Score > +30: buy at +10% premium
  - Score < -20: refuse to trade

### 7. Show reputation in CLI

Add to SectorScreen NPC list:
```
📦 Kira the Trader — friendly (reputation: +45)
⚠️ Void Fang — hostile (reputation: -72)
```

Color-code: green = positive, red = negative, gray = neutral.

### 8. DB migration for memory

`NPCMemory` gains a `reputation` field. Since NPCs are stored as JSON (`npcsJson`), no schema migration needed — just add the field and default to `{}` on load.

## Notes
- **Memory is per-NPC, not shared.** NPCs don't gossip (yet). Each has their own opinion of you.
- **Reputation is asymmetric.** You can love a trader who hates you (if you attacked them but they don't know it was you... but they do, because combat reveals identity).
- **Grudges are strong, alliances are fragile.** Easier to make an enemy than a friend. Matches human psychology and game tension.
