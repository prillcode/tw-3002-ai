# 03-05-PLAN — Response Caching

## Objective
Skip redundant LLM calls by caching decisions. If an NPC is in the same sector with the same stats and no new events, don't ask the LLM again — reuse the previous answer. This cuts API costs by 50-80% and makes the game feel snappier.

## Context
- TW-03 Phase 2 has async `decideAction()` that calls the LLM provider every time
- 15 active NPCs × 1 tick per session = 15 LLM calls minimum
- Many NPCs are in stable situations: a trader at a port with no player nearby will likely make the same decision repeatedly
- PRD cost target: ~$0.025/session. Without caching, 15 calls × ~500 tokens × $0.0005/1K tokens = $0.00375. Caching could halve this.
- `MockProvider` is deterministic, but Ollama/OpenRouter are not (temperature adds variance)

## Approach

Cache key = hash of the inputs that determine a decision:
```
cacheKey = hash(npc.persona.type + currentSector.danger + playerPresent + hullPct + shieldPct + cargoEmpty + hasGrudgeVsPlayer + lastActionType)
```

**Cache rules:**
- Hit → return cached action instantly, skip LLM call
- Miss → call LLM, store result, return action
- TTL: 30 seconds (volatile situations change fast) OR 1 tick (if we only tick once per login, TTL is irrelevant)
- Max size: 100 entries (small, per-session memory)

**When NOT to cache:**
- Player is in the same sector (volatile — player might have just attacked/traded)
- NPC hull < 50% (desperate situations need fresh reasoning)
- NPC has a new grudge (emotional state changed)

## Tasks

### 1. Create cache module (`packages/engine/src/llm/cache.ts`)

```typescript
export interface CacheEntry {
  action: NPCAction;
  reasoning: string;
  timestamp: number;  // Date.now()
}

export class DecisionCache {
  private cache = new Map<string, CacheEntry>();
  private maxSize: number;
  private ttlMs: number;

  constructor(maxSize = 100, ttlMs = 30000) {
    this.maxSize = maxSize;
    this.ttlMs = ttlMs;
  }

  get(key: string): CacheEntry | null;
  set(key: string, entry: CacheEntry): void;
  has(key: string): boolean;
  clear(): void;
  getStats(): { hits: number; misses: number; size: number };
}
```

**LRU eviction:** When cache exceeds `maxSize`, evict oldest entry.

**Key generation:**
```typescript
function makeCacheKey(npc: NPC, galaxy: Galaxy, players: PlayerShip[]): string {
  const sector = galaxy.sectors.get(npc.currentSectorId);
  const playerHere = players.some(p => p.currentSector === npc.currentSectorId);
  const hullPct = Math.round((npc.ship.hull / npc.ship.maxHull) * 10) / 10; // 1 decimal
  const shieldPct = Math.round((npc.ship.shield / npc.ship.maxShield) * 10) / 10;
  const cargoTotal = Object.values(npc.cargo).reduce((a, b) => a + b, 0);
  const hasGrudge = npc.memory.grudges.some(g => g.targetId === 'player');

  return [
    npc.persona.type,
    sector?.danger ?? 'unknown',
    playerHere ? 'player-yes' : 'player-no',
    `hull-${hullPct}`,
    `shield-${shieldPct}`,
    `cargo-${cargoTotal}`,
    hasGrudge ? 'grudge-yes' : 'grudge-no',
    npc.memory.lastActions[0]?.action.type ?? 'none',
  ].join('|');
}
```

### 2. Integrate cache into `decideAction()`

Modify `packages/engine/src/npcs/brain.ts`:

```typescript
const globalCache = new DecisionCache(); // module-level singleton

export async function decideAction(
  npc: NPC,
  galaxy: Galaxy,
  players: PlayerShip[],
  config?: LLMConfig,
): Promise<NPCAction> {
  // Fast path: no config or disabled → rules (no caching needed)
  if (!config || config.provider === 'disabled') {
    return decideRuleBased(npc, galaxy, players);
  }

  // Should we use cache?
  const shouldCache = canUseCache(npc, players);
  const cacheKey = shouldCache ? makeCacheKey(npc, galaxy, players) : null;

  if (cacheKey) {
    const cached = globalCache.get(cacheKey);
    if (cached) {
      // Record in memory that this was a cached decision
      npc.memory.lastActions.push({
        action: cached.action,
        result: `Cached: ${cached.reasoning}`,
        at: new Date().toISOString(),
      });
      return cached.action;
    }
  }

  // Call LLM (existing code)
  const provider = createProvider(config);
  // ... prompt, call, parse ...

  if (parsed && cacheKey) {
    globalCache.set(cacheKey, {
      action: parsed.action,
      reasoning: parsed.reasoning,
      timestamp: Date.now(),
    });
  }

  return parsed?.action ?? decideRuleBased(npc, galaxy, players);
}
```

```typescript
function canUseCache(npc: NPC, players: PlayerShip[]): boolean {
  // Don't cache if player is in same sector (volatile)
  if (players.some(p => p.currentSector === npc.currentSectorId)) return false;
  // Don't cache if hull is low (desperate)
  if (npc.ship.hull / npc.ship.maxHull < 0.5) return false;
  // Don't cache if new grudge formed recently (emotional)
  const recentGrudge = npc.memory.grudges.some(g => {
    const age = Date.now() - new Date(g.formedAt).getTime();
    return age < 60000; // 1 minute
  });
  if (recentGrudge) return false;
  return true;
}
```

## Verification

1. **Cache hit** — Same NPC in same situation → second call returns cached action instantly (no network delay)
2. **Cache miss (player nearby)** — Player enters sector → cache bypassed → fresh LLM call
3. **Cache miss (low hull)** — NPC hull drops below 50% → cache bypassed → fresh LLM call
4. **Cache eviction** — After 100 unique situations, oldest cache entry removed
5. **Stats tracking** — After 10 calls, stats show accurate hit/miss ratio
6. **Mock provider** — Even with mock (deterministic), caching still works and shows hits
7. **TypeScript** — `tsc --noEmit` clean

## Done When

## Notes
- **Cache is a session-level optimization.** It lives in memory, dies when the process exits. That's fine — each login is a new session with fresh state.
- **Cache key coarseness:** We round hull to 1 decimal (10% buckets) so minor damage doesn't bust the cache, but major damage does. Fine-tune if needed.
- **Temperature interaction:** If temperature > 0, the LLM might return different answers for the same prompt. Caching still helps because many NPCs in similar situations get the *same* prompt key. The variance is acceptable — we want consistency for identical situations anyway.
