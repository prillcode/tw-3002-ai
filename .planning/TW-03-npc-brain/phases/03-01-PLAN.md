# 03-01-PLAN — NPC Brain Foundation

## Objective
Lay the groundwork for LLM-driven NPCs: create the NPC data model, a lightweight decision pipeline, and a single "dumb but functional" NPC type that moves and trades in the galaxy. No real LLM calls yet — use rule-based fallbacks so the system works offline and cheaply. The LLM layer plugs in later.

## Context
- TW-03 BRIEF defines the vision: NPCs with personas, memory, and LLM reasoning
- PRD Section 4.2: Dormant galaxy (only 2-3 sector radius active), ~$0.025/session cost target
- TW-02 provides the game engine: galaxy, economy, combat types
- Current state: zero NPC code exists
- Risk: LLM costs and latency can kill the experience. Build the fallback-first.

## Approach

Build NPCs in three layers:
1. **Data layer** — NPC types, personas, memory structures
2. **Decision layer** — rule-based action selection (works offline, $0 cost)
3. **LLM layer** — prompt templates, OpenRouter integration, response parsing (future plan)

This plan covers layers 1 and 2. Layer 3 is 03-02.

## Tasks

### 1. Define NPC types and personas (`packages/engine/src/npcs/types.ts`)

```typescript
export type NPCType = 'trader' | 'raider' | 'patrol';

export interface NPCPersona {
  type: NPCType;
  name: string;
  aggression: number;      // 0-1, likelihood to attack
  caution: number;         // 0-1, likelihood to flee
  greed: number;           // 0-1, prioritizes profit
  loyalty: number;         // 0-1, faction stickiness
  flavor: string;          // brief personality description
}

export interface NPCMemory {
  lastActions: NPCAction[];     // last 3 turns
  grudges: Grudge[];
  alliances: Alliance[];
  marketObservations: MarketObservation[];
}

export interface NPC {
  id: string;
  persona: NPCPersona;
  ship: Combatant;              // reuses combat types
  currentSectorId: number;
  credits: number;
  cargo: Record<Commodity, number>;
  memory: NPCMemory;
  isActive: boolean;            // awake vs dormant
}

export type NPCAction =
  | { type: 'move'; targetSector: number }
  | { type: 'trade'; trade: TradeAction }
  | { type: 'attack'; targetId: string }
  | { type: 'flee'; targetId: string }
  | { type: 'idle' };
```

### 2. Create NPC generator (`packages/engine/src/npcs/generator.ts`)

```typescript
export function generateNPCs(galaxy: Galaxy, count: number): NPC[];
```

- Generate 20-30 NPCs per galaxy
- Distribute by type: 50% trader, 30% raider, 20% patrol
- Place initially in random sectors
- Patrols cluster near FedSpace; raiders in dangerous sectors; traders everywhere
- Names: prefix + suffix pools (like sectors but character-flavored)
- Starting credits: 2,000-10,000 based on type
- Starting cargo: small random amounts

### 3. Create rule-based decision engine (`packages/engine/src/npcs/brain.ts`)

```typescript
export function decideAction(npc: NPC, galaxy: Galaxy, nearbyPlayers: PlayerShip[]): NPCAction;
```

**Trader logic:**
1. If in sector with port AND profitable trade exists → `trade`
2. If cargo full AND no good sell price here → `move` toward port that buys high
3. If low on credits → `move` toward safe sector
4. Otherwise → `idle` or random `move`

**Raider logic:**
1. If player in same sector AND aggression roll passes → `attack`
2. If player in same sector AND caution roll passes → `flee` (weak raiders)
3. If no player → `move` toward dangerous/frontier sectors
4. If hull < 30% → `move` toward safe sector to "hide"

**Patrol logic:**
1. If raider in same sector → `attack`
2. Otherwise → `move` randomly within FedSpace radius

**Simple pathfinding:**
- BFS to find route to target sector
- Move one step toward target per turn
- If no route, random legal move

### 4. Integrate NPCs into GameState

Add to `GameState`:
```typescript
npcs: NPC[];
```

Add to `GameStateContainer`:
- `getNPCsInSector(sectorId): NPC[]`
- `getActiveNPCs(playerSectorId, radius): NPC[]` — dormant galaxy filter
- `tickNPCs(): GameStateContainer` — process one turn for all active NPCs

**Dormant galaxy logic:**
- Only NPCs within 2 sectors of any player are "active"
- Active NPCs take turns (rule-based for now, LLM later)
- Dormant NPCs are frozen — no state changes
- On player login: recompute active set, tick active NPCs once

### 5. Add NPCs to CLI SectorScreen

- Show NPC ships in sector list: "2 ships present"
- Color-code: traders = cyan, raiders = red, patrols = green
- Show NPC names in SectorInfo panel
- If raider present, warn: "⚠️ Pirate 'Void Fang' spotted!"

### 6. Add NPCs to combat system

- `Combatant` already supports name, hull, weapons, etc.
- Convert NPC to Combatant when encounter triggers
- After combat, update NPC state (hull, credits, cargo)
- If NPC destroyed: remove from galaxy, generate replacement after N turns

### 7. Add NPC save/load

- NPCs serialize as JSON array in save
- On load: reconstruct NPCs with their memory intact
- NPC memory gives continuity — "that trader remembers you cheated them"

### 8. Add a "Galaxy News" feed

- Track notable NPC actions: "Pirate raids reported in Sector 67", "Trader convoy arrives at StarDock"
- Show 3-5 recent headlines on SectorScreen or a dedicated NewsScreen (N key)
- Generated from NPC action log, not LLM-generated prose (keep it cheap)

## Verification

1. **Generator** — `generateNPCs(galaxy, 20)` creates 20 valid NPCs, no duplicates
2. **Trader** — Trader in sector with port executes at least 1 trade per 3 turns
3. **Raider** — Raider in same sector as player initiates combat within 5 turns
4. **Patrol** — Patrol stays within FedSpace radius
5. **Dormant** — NPCs outside 2-sector radius don't take actions
6. **Save/load** — NPC positions and memories persist across sessions
7. **News** — Headlines appear after NPC actions
8. **Performance** — 20 NPCs tick in < 10ms (rule-based is fast)
9. **CLI** — `bun run typecheck` + `bun run build` clean

- Traders move between ports and trade

- Patrols guard FedSpace

## Notes
- **No LLM yet.** This phase proves the NPC system works with cheap rules. LLM layer (03-02) swaps in later.
- **NPC memory is structural, not semantic.** Last 3 actions + grudges + alliances. No free-text memory yet.
- **NPC cargo/economy:** NPCs participate in the same economy as players. Their trades affect port prices. This makes the economy feel alive.
- **Balancing:** Start with 20 NPCs. Tune density based on galaxy size.
- **Replacement:** Destroyed NPCs respawn after 10-20 player turns to keep galaxy populated.
