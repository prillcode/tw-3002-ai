# 03-03-PLAN — NPC Ticks & Dormant Galaxy

## Objective
Make NPCs actually "live." When a player logs in, all NPCs within a 2-sector radius wake up, take turns (move, trade, attack), and go back to sleep. This is the dormant galaxy: only ~10-15 NPCs activate per session, keeping costs minimal. The galaxy feels alive because NPCs evolve between your visits.

## Context
- TW-03 Phase 1 has 20 NPCs per galaxy, displayed in sectors, with rule-based + LLM brains
- Phase 2 has pluggable LLM providers (Ollama/OpenRouter/Mock) — but NPCs never actually *use* them
- Current state: NPCs are frozen decorations. They show up in sectors and can attack when you jump in, but they never move or trade on their own
- PRD Section 4.2: "NPCs take their turns when players log in. The universe is paused between sessions."
- `GameStateContainer.getActiveNPCs(radius=2)` already exists — just not used

## Approach

```
Player logs in → load game state
  → identify active NPCs (within 2 sectors of player)
  → for each active NPC:
      → decideAction() (LLM if configured, else rules)
      → executeNPCAction() (mutates galaxy: port inventory, NPC position, NPC cargo/credits)
      → record news item
  → NPCs outside radius stay frozen
  → save state
  → player begins their turn
```

**Turn order per NPC:**
1. **Trader:** If at port → try trade. If cargo full + no good prices → move toward port. If credits low → move toward safe sector.
2. **Raider:** If player in sector + aggression roll → attack. If low hull → flee to safe sector. Otherwise → prowl dangerous sectors.
3. **Patrol:** If raider in sector → attack. Otherwise → patrol FedSpace.

**Performance target:** 15 NPCs × decideAction (rule-based = <1ms, LLM = 500-3000ms) = acceptable. Process serially, not in parallel, to avoid overwhelming local Ollama.

## Tasks

### 1. Create `tickNPCs()` function (`packages/engine/src/npcs/tick.ts`)

```typescript
export async function tickNPCs(
  container: GameStateContainer,
  llmConfig?: LLMConfig,
): Promise<{ container: GameStateContainer; news: NewsItem[]; stats: TickStats }>;

interface TickStats {
  npcsProcessed: number;
  actionsTaken: number;
  llmCalls: number;
  llmCost: number;
  durationMs: number;
}
```

Implementation:
1. Get active NPCs via `container.getActiveNPCs(container.currentSectorId, 2)`
2. For each active NPC:
   - Call `decideAction(npc, galaxy, [player], llmConfig)` (async)
   - Call `executeNPCAction(npc, action, galaxy)`
   - Collect news items
   - Update NPC in array
3. Return new container with mutated NPCs + galaxy + news
4. Track stats for display

**Shuffling:** Randomize NPC turn order so no one NPC always acts first.

### 2. Wire `tickNPCs()` into game flow

**Trigger points:**
- **On new game:** After ship class selection, before showing sector → tick once to "seed" the galaxy
- **On continue:** After loading save, before showing sector → tick once
- **After player logout → login:** This is the main one — the universe evolves while you were away

**For now (local single-player):** Tick on every sector jump is too much. Tick once at the start of the session (continue/new game) and then leave NPCs frozen during the session. Future: tick after every player turn.

In `cli/src/index.tsx`:
```typescript
// After loadExistingGame() completes:
const { tickNPCs } = require('@tw3002/engine');
const result = await tickNPCs(gameContainer, llmConfig);
setNpcs(result.container.npcs);
setNews(prev => [...prev, ...result.news]);
// Apply any galaxy mutations (port inventory changes from NPC trades)
setGalaxy(result.container.galaxy);
```

Add this as a brief panel on the SectorScreen (only on first render after login, not every sector view).

### 4. Handle NPC-to-NPC combat

When a raider and a trader are in the same sector during the tick:
- Raider's `decideAction()` sees the trader
- May choose `attack`
- `executeNPCAction()` resolves the fight with simplified rules (no combat screen, just dice roll)
- Loser takes hull damage, may flee or be "destroyed" (removed, respawn later)
- Winner gains credits/cargo
- News item generated

**Simplified NPC combat resolver:**
```typescript
function resolveNPCCombat(attacker: NPC, defender: NPC): { winner: NPC; loser: NPC; news: NewsItem }
```
- Compare weaponDamage + random(0.8-1.2) vs hull
- 3 rounds max
- Loser hull <= 0 → "destroyed" (flag for respawn, don't remove immediately)

### 5. Handle NPC respawning

If an NPC is "destroyed" during a tick:
- Don't remove from array yet (would break indices in save)
- Set `npc.ship.hull = 0`, `npc.isActive = false`
- After the tick, replace destroyed NPCs with new ones
- New NPC spawns at a random sector appropriate to their type
- News item: "A new trader arrives in the galaxy"

### 6. Add tick stats display

After ticking, show a brief summary:
```
Galaxy evolved: 12 NPCs took actions (2 trades, 5 moves, 1 fight, 4 idle)
LLM cost this session: $0.003
```

This could be a small panel on SectorScreen or a one-time splash after login.

## Verification

1. **Rule-based tick** — Start new game, 15 active NPCs tick in < 50ms total
2. **LLM tick** — With Mock provider, 15 NPCs tick and produce deterministic actions
3. **Port inventory changes** — NPC traders buy/sell → port stock changes → player sees different prices
4. **NPC movement** — After tick, some NPCs are in different sectors than where they spawned
5. **News generation** — At least 3-5 news items generated per tick
6. **Save/load** — NPC positions and states persist across sessions
7. **No crash on empty galaxy** — tick works even if all NPCs destroyed
8. **TypeScript** — `tsc --noEmit` clean
9. **Binary build** — `bun run build` succeeds

## Done When
- NPCs process turns on every player login
- Active NPCs (2-sector radius) move, trade, and fight
- NPC actions mutate galaxy state (port inventory, positions)
- News feed shows what happened while player was away
- Destroyed NPCs respawn to keep population at ~20
- LLM costs tracked per session and displayed
- Rule-based mode: zero cost, instant
- TypeScript clean, binary builds

## Notes
- **Session scope:** One tick per login is enough for the local MVP. Don't tick on every jump — that's overkill and would make the game feel sluggish with LLM.
- **NPC-to-NPC combat is simplified** — no combat screen, just dice rolls. Players only see the news headline.
- **Port economy:** NPC trades are small (5-10 units) so they don't wreck the economy, but over many ticks they create visible price drift.
