# 03-02-PLAN — Provider-Agnostic LLM Integration

## Objective
Add LLM reasoning to NPC decisions without locking into a single provider. Build a pluggable client that supports local (Ollama) and cloud (OpenRouter) providers, with a config file so users choose their own backend. Rule-based brain remains the default fallback — LLM is opt-in.

## Context
- TW-03 Phase 1 (NPC Brain Foundation) is complete — rule-based NPCs exist, move, trade, fight
- PRD Section 7.1 defines the LLM provider strategy
- NPC brain currently lives in `packages/engine/src/npcs/brain.ts` — `decideAction()` is pure rules
- The project must work offline by default; LLM is an enhancement, not a requirement
- In local mode, the user bears LLM costs (their API key or their GPU). In cloud mode (future TW-04), the host pays.

## Approach

Build three layers:
1. **Provider interface** — abstract LLM client (prompt in, structured JSON out)
2. **Provider implementations** — Ollama (local), OpenRouter (cloud), Mock (dev/testing)
3. **Decision pipeline** — `decideAction()` checks config, uses LLM if available, falls back to rules

```
Config (~/.tw3002/config.json)
  │
  ├── "disabled" → rule-based brain (default)
  ├── "ollama"   → POST http://localhost:11434/api/chat
  ├── "openrouter" → POST https://openrouter.ai/api/v1/chat/completions
  └── "mock"     → deterministic test responses

DecideAction(npc, galaxy, players, config)
  │
  ├── No config / disabled → rule-based decision (Phase 1)
  └── LLM enabled → build prompt → call provider → parse JSON → validate → apply
```

## Tasks

### 1. Create provider interface (`packages/engine/src/llm/provider.ts`)

```typescript
export interface LLMProvider {
  readonly name: string;
  chat(messages: ChatMessage[], options?: ChatOptions): Promise<ChatResponse>;
}

export interface ChatMessage {
  role: 'system' | 'user';
  content: string;
}

export interface ChatOptions {
  temperature?: number;
  maxTokens?: number;
  responseFormat?: 'json';
}

export interface ChatResponse {
  content: string;
  usage?: { promptTokens: number; completionTokens: number };
  model?: string;
}

export interface LLMConfig {
  provider: 'disabled' | 'ollama' | 'openrouter' | 'mock';
  model?: string;
  apiKey?: string;
  endpoint?: string;       // override default (e.g. custom Ollama URL)
  temperature?: number;
  maxTokens?: number;
}
```

### 2. Create Ollama provider (`packages/engine/src/llm/ollama.ts`)

- `OllamaProvider` implements `LLMProvider`
- Default endpoint: `http://localhost:11434/api/chat`
- Default model: `llama3.2:3b` (3B params, fast, decent reasoning)
- Request body: `{ model, messages, stream: false, format: 'json' }`
- Response parsing: extract `message.content` JSON string
- Error handling: connection refused → return `null` (fallback to rules)
- No API key needed

**Verify Ollama is running:**
```typescript
async function isOllamaAvailable(url: string): Promise<boolean>
```

### 3. Create OpenRouter provider (`packages/engine/src/llm/openrouter.ts`)

- `OpenRouterProvider` implements `LLMProvider`
- Endpoint: `https://openrouter.ai/api/v1/chat/completions`
- Default model: `openai/gpt-4o-mini`
- Headers: `Authorization: Bearer {apiKey}`, `HTTP-Referer`, `X-Title`
- Request body: OpenAI-compatible `{ model, messages, temperature, max_tokens, response_format: { type: 'json_object' } }`
- Response parsing: extract `choices[0].message.content`
- Error handling: 401 (bad key), 429 (rate limit), 5xx → retry once, then fallback
- Cost tracking: read `usage` from response, accumulate per session

### 4. Create Mock provider (`packages/engine/src/llm/mock.ts`)

- `MockProvider` implements `LLMProvider`
- Returns deterministic responses based on prompt hash
- No network calls — instant, free, predictable for testing
- Define 10-15 canned decisions per NPC type
- Used in CI and for testing prompt changes without burning API credits

### 7. Create response parser (`packages/engine/src/llm/parser.ts`)

```typescript
export function parseNPCDecision(json: string): {
  action: NPCAction;
  reasoning: string;
} | null;
```

- Parse JSON from LLM response
- Validate action type against allowed values
- Validate params against schema (required fields, types, ranges)
- Validate action is legal in current state (e.g., `move` target must be a neighbor)
- Return `null` on any validation failure → triggers fallback to rule-based
- Catch and log parse errors for debugging

### 8. Wire LLM into `decideAction()` (`packages/engine/src/npcs/brain.ts`)

Replace the current rule-based `decideAction` with a hybrid:

```typescript
export async function decideAction(
  npc: NPC,
  galaxy: Galaxy,
  players: PlayerShip[],
  config?: LLMConfig,
): Promise<NPCAction> {
  // 1. Fast path: no config or disabled → rules
  if (!config || config.provider === 'disabled') {
    return decideRuleBased(npc, galaxy, players);
  }

  // 2. Build provider
  const provider = createProvider(config);
  if (!provider) return decideRuleBased(npc, galaxy, players);

  // 3. Build prompt
  const prompt = buildPrompt(npc, galaxy, players);

  // 4. Call LLM (with timeout)
  try {
    const response = await withTimeout(
      provider.chat([
        { role: 'system', content: SYSTEM_PROMPT },
        { role: 'user', content: prompt },
      ], { temperature: config.temperature, maxTokens: config.maxTokens, responseFormat: 'json' }),
      5000 // 5 second timeout
    );

    // 5. Parse and validate
    const parsed = parseNPCDecision(response.content);
    if (parsed) {
      // Record reasoning in NPC memory (for flavor, not logic)
      // ...
      return parsed.action;
    }
  } catch (err) {
    // Log error, fallback to rules
  }

  // 6. Fallback
  return decideRuleBased(npc, galaxy, players);
}
```

**Rename current `decideAction` to `decideRuleBased`** — keep it intact as the fallback.

## Notes
- **No LLM by default.** The out-of-box experience is rule-based (free, offline, fast). Users opt into LLM via config.
- **Ollama is the recommended local path.** Free, private, no API keys. User installs Ollama + pulls a model.
- **OpenRouter is the recommended cloud path.** One key, multiple models, competitive pricing. Good for users who want "smarter" NPCs without running a local model.
- **Mock provider is for devs.** Use it to test prompt changes and decision logic without burning credits.
- **Cost target:** With response caching (Phase 6 in roadmap) and dormant galaxy (Phase 5), active NPCs should cost <$0.05/session even with OpenRouter.
- **Async decisions:** `decideAction()` becomes `async` because LLM calls are network-bound. The caller (`tickNPCs()` or CLI) must `await` it.
