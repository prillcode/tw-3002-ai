# 11-01 — Anti-Spam Foundation

## Objective
Add bot/abuse protections to the auth system before enabling email verification. Verified accounts are only valuable if they belong to real people.

## Context
- Current `/api/auth/register` accepts any email string, generates a UUID token immediately
- No validation on email format, no blocklist on disposable domains
- No bot protection on the web client login form
- Rate limiting exists on auth endpoints (5/min/IP) from TW-12, but that's the only defense
- The `players` table already accumulates every email entered — we need to keep that list clean

## Design Decisions

### Disposable Email Blocklist
- Use a curated blocklist of ~3,000 known disposable/throwaway domains
- Check at registration time — reject with 400 if domain matches
- Store blocklist as a static Set in `cloud/src/utils/email.ts` (compiled at deploy time)
- Refresh the blocklist periodically (manual update or CI script)

### Email Format Validation
- Regex for basic structure: `user@domain.tld`
- Minimum domain length (e.g. no `a@b.c`)
- Optional: MX record lookup (can be slow; skip for MVP)

### Turnstile (Invisible CAPTCHA)
- Cloudflare Turnstile — free, no visual challenge, privacy-focused
- Add to the Vue login/registration form
- Verify Turnstile token server-side before processing register
- Falls back gracefully if Turnstile is unavailable

### Registration Tracking
- Add `registration_ip` and `registration_attempts` tracking (lightweight)
- Already covered by TW-12 rate limiting — no extra DB columns needed

## Tasks

### 1. Create `cloud/src/utils/email.ts`

```typescript
const DISPOSABLE_DOMAINS = new Set([
  '10minutemail.com',
  'guerrillamail.com',
  'mailinator.com',
  // ... ~3,000 entries
]);

export function isValidEmail(email: string): boolean {
  const re = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!re.test(email)) return false;
  const parts = email.split('@');
  if (parts.length !== 2) return false;
  const domain = parts[1].toLowerCase();
  if (domain.length < 4) return false; // a.bc is too short
  return true;
}

export function isDisposableDomain(domain: string): boolean {
  return DISPOSABLE_DOMAINS.has(domain.toLowerCase());
}
```

Generate the blocklist from a public source (e.g. `disposable-email-domains` npm package or GitHub repo).

### 2. Update `cloud/src/routes/auth.ts` — register validation

Before creating the player record:
```typescript
if (!isValidEmail(email)) return jsonError('Invalid email format', 400);
const domain = email.split('@')[1];
if (isDisposableDomain(domain)) return jsonError('Disposable email addresses are not allowed', 400);
```

### 3. Add Turnstile to the Vue login form

In `web/game/src/views/LoginView.vue`:
- Add `<script src="https://challenges.cloudflare.com/turnstile/v0/api.js" async defer></script>`
- Add Turnstile widget: `<div class="cf-turnstile" data-sitekey="YOUR_SITE_KEY"></div>`
- Capture the Turnstile token and send it with the register request

### 4. Verify Turnstile server-side

In `handleRegister`, before processing:
```typescript
const turnstileToken = body.turnstileToken;
if (!turnstileToken) return jsonError('Turnstile verification required', 400);

const verifyRes = await fetch('https://challenges.cloudflare.com/turnstile/v0/siteverify', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    secret: env.TURNSTILE_SECRET_KEY,
    response: turnstileToken,
  }),
});
const verifyData = await verifyRes.json();
if (!verifyData.success) return jsonError('Turnstile verification failed', 400);
```

### 5. Add `TURNSTILE_SECRET_KEY` to Worker secrets

```bash
cd cloud && wrangler secret put TURNSTILE_SECRET_KEY
```

Update `Env` interface in `cloud/src/index.ts`:
```typescript

```

No changes needed for Turnstile site key (it's public, used in HTML).
