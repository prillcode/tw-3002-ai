# 11-02 — Email Verification Flow

## Objective
Implement OTP-based email verification so players must verify their email before creating a ship or playing the game.

## Context
- Current flow: `register` → immediate UUID token → play immediately
- Target flow: `register` → email sent with 6-digit OTP → `verify-email` with OTP → bearer token → play
- The `players` table needs new columns for verification state
- Need an email provider for transactional emails (Resend recommended — free tier, great DX)
- Vue login flow needs a "check your email" state

## Design Decisions

### OTP over Magic Link
- **OTP (6-digit code)** is chosen over magic links because:
  - Works better in terminal/CLI clients (users can type a code)
  - No need to handle deep links or browser tabs
  - Simpler UX in the web client (one form field)
  - Shorter email content = better deliverability

### Email Provider: Resend
- Free tier: 3,000 emails/month
- Simple REST API
- Good deliverability
- No credit card required for free tier
- Alternative: SendGrid (more complex setup)

### Token Flow
1. `POST /api/auth/register` { email, turnstileToken }
   - Creates player row with `email_verified = 0`
   - Generates 6-digit OTP, stores hash in `verification_token`
   - Sets `verification_expires_at` (+15 minutes)
   - Sends email with OTP via Resend
   - Returns `{ success: true, message: "Check your email for a verification code" }`

2. `POST /api/auth/verify-email` { email, otp }
   - Looks up player by email
   - Checks OTP against stored hash
   - Checks expiry
   - Sets `email_verified = 1`
   - Generates bearer token (UUID)
   - Returns `{ success: true, authToken, expiresAt }`

3. `POST /api/player/ship` (create ship)
   - Checks `email_verified = 1` before allowing ship creation
   - Returns 403 if not verified: `{ error: "Email verification required" }`

### OTP Generation
```typescript
function generateOTP(): string {
  return Math.floor(100000 + Math.random() * 900000).toString();
}
```

Store the SHA-256 hash of the OTP (not the plaintext) in the database for security.

```typescript
export async function sendVerificationEmail(
  apiKey: string,
  to: string,
  otp: string,
): Promise<{ success: boolean; error?: string }> {
  const res = await fetch('https://api.resend.com/emails', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      from: 'TW 3002 AI <noreply@playtradewars.net>',
      to,
      subject: 'Your verification code for TW 3002 AI',
      html: `<p>Your verification code is:</p><h1>${otp}</h1><p>This code expires in 15 minutes.</p>`,
      text: `Your verification code is: ${otp}\nThis code expires in 15 minutes.`,
    }),
  });

  if (!res.ok) {
    const err = await res.text();
    return { success: false, error: err };
  }
  return { success: true };
}
```

### 4. Rewrite `cloud/src/routes/auth.ts`

**New `handleRegister`:**
```typescript
export async function handleRegister(
  request: Request,
  db: D1Database,
  env: { TURNSTILE_SECRET_KEY?: string; RESEND_API_KEY?: string }
): Promise<Response> {
  // ... validation (email format, disposable domain, turnstile) ...

  const otp = generateOTP();
  const otpHash = await sha256(otp);
  const expiresAt = new Date(Date.now() + 15 * 60 * 1000).toISOString();

  // Upsert player (unverified)
  const existing = await db.prepare('SELECT id FROM players WHERE email = ?').bind(email).first();
  if (existing) {
    await db.prepare(
      'UPDATE players SET verification_token = ?, verification_expires_at = ?, verification_attempts = 0, email_verified = 0 WHERE id = ?'
    ).bind(otpHash, expiresAt, existing.id).run();
  } else {
    await db.prepare(
      'INSERT INTO players (email, verification_token, verification_expires_at, email_verified, created_at) VALUES (?, ?, ?, 0, datetime("now"))'
    ).bind(email, otpHash, expiresAt).run();
  }

  // Send email
  if (env.RESEND_API_KEY) {
    await sendVerificationEmail(env.RESEND_API_KEY, email, otp);
  } else {
    console.log(`[DEV] Verification code for ${email}: ${otp}`);
  }

  return json({ success: true, message: 'Check your email for a verification code' });
}
```

**New `handleVerifyEmail`:**
```typescript
export async function handleVerifyEmail(
  request: Request,
  db: D1Database
): Promise<Response> {
  let body: { email?: string; otp?: string };
  try { body = await request.json(); } catch { return jsonError('Invalid JSON'); }

  const { email, otp } = body;
  if (!email || !otp) return jsonError('email and otp required');

  const player = await db.prepare(
    'SELECT id, verification_token, verification_expires_at, verification_attempts FROM players WHERE email = ?'
  ).bind(email).first<{
    id: number;
    verification_token: string | null;
    verification_expires_at: string | null;
    verification_attempts: number;
  }>();

  if (!player) return jsonError('Invalid email', 401);
  if (player.verification_attempts >= 5) return jsonError('Too many attempts. Request a new code.', 429);

  // Increment attempts
  await db.prepare('UPDATE players SET verification_attempts = verification_attempts + 1 WHERE id = ?').bind(player.id).run();

  // Check expiry
  if (player.verification_expires_at && new Date(player.verification_expires_at) < new Date()) {
    return jsonError('Verification code expired', 401);
  }

  // Check OTP
  const otpHash = await sha256(otp);
  if (player.verification_token !== otpHash) {
    return jsonError('Invalid verification code', 401);
  }

  // Success — generate bearer token
  const authToken = crypto.randomUUID();
  const tokenExpiresAt = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000).toISOString();

  await db.prepare(
    'UPDATE players SET email_verified = 1, auth_token = ?, token_expires_at = ?, verification_token = NULL, verification_expires_at = NULL, last_login_at = datetime("now") WHERE id = ?'
  ).bind(authToken, tokenExpiresAt, player.id).run();

  return json({ success: true, authToken, expiresAt: tokenExpiresAt });
}
```
