# Phase 01 Plan: Daily Bounty System — Server

**Work Item:** TW-19 — Leaderboards & Daily Bounties  
**Phase:** 01 — Daily Bounty System: Server  
**Estimated Time:** 2–3 hours  
**Prerequisite:** None (builds on existing action endpoints)

---

## Objective
Build the server-side daily mission system: database schema, mission generation, progress tracking hooks, claim/reward endpoints, and UTC midnight reset.

---

## Context

The game has a solid core loop (move, trade, fight, claim planets) but lacks structured daily goals. We need to hook into existing action endpoints to increment mission progress without cluttering every handler.

**Existing architecture we leverage:**
- `player_ships` table holds per-galaxy player state
- `checkAndDeductActionPoints()` runs before every gameplay action
- Combat (`handleCombat`), trade (`handleTrade`), move (`handleMoveShip`), planet colonize (`handleColonize`), and tax payment (`handlePayTaxes`) are the key action endpoints
- Visited sectors are currently client-side only (`galaxy.ts: visitedIds`) — we need server-side tracking for missions

**Key design decision:** A single `trackMissionProgress()` utility called from action handlers after successful completion. No middleware, no wrappers — explicit calls at the point of success.

---

## Database Schema

### Migration: `0012_daily_missions.sql`

```sql
-- Daily missions per player per galaxy
CREATE TABLE IF NOT EXISTS player_daily_missions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  player_id INTEGER NOT NULL,
  galaxy_id INTEGER NOT NULL,
  mission_type TEXT NOT NULL,       -- 'kill_npcs', 'trade_credits', 'visit_sectors', 'claim_planet', 'pay_taxes'
  target_count INTEGER NOT NULL DEFAULT 1,
  current_count INTEGER NOT NULL DEFAULT 0,
  reward_credits INTEGER NOT NULL DEFAULT 0,
  completed INTEGER NOT NULL DEFAULT 0,   -- 1 when current_count >= target_count
  claimed INTEGER NOT NULL DEFAULT 0,     -- 1 when reward claimed
  claimed_at DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  mission_date TEXT NOT NULL,       -- 'YYYY-MM-DD' for UTC reset
  UNIQUE(player_id, galaxy_id, mission_type, mission_date)
);

CREATE INDEX IF NOT EXISTS idx_missions_player_galaxy_date ON player_daily_missions(player_id, galaxy_id, mission_date);

-- Track visited sectors server-side for mission progress
ALTER TABLE player_ships ADD COLUMN visited_sectors_json TEXT DEFAULT '[]';
```

### Task 2: Create mission utilities module
**File:** `cloud/src/utils/dailyMissions.ts`

**Functions to implement:**

#### `generateDailyMissions(db, playerId, galaxyId): Promise<Mission[]>`
- Check if missions already exist for today (`mission_date = date('now')`)
- If yes, return them
- If no, generate 3 missions (weighted: 1 easy, 1 medium, 1 hard)
- Insert into `player_daily_missions`
- Return the generated missions

**Mission types & difficulty pools:**

| Type | Easy | Medium | Hard |
|---|---|---|---|
| `kill_npcs` | Kill 2–3 NPCs (500–1,000cr) | Kill 5–8 NPCs (1,500–2,500cr) | Kill 10–15 NPCs (3,000–5,000cr) |
| `trade_credits` | Earn 3,000–5,000cr trading (500–800cr) | Earn 10,000–20,000cr (1,500–2,500cr) | Earn 30,000–50,000cr (3,000–5,000cr) |
| `visit_sectors` | Visit 3–5 new sectors (400–700cr) | Visit 8–12 new sectors (1,000–1,800cr) | Visit 15–20 new sectors (2,000–3,500cr) |
| `claim_planet` | — | Colonize 1 planet (1,500–2,500cr) | Colonize 2 planets (3,000–4,000cr) |
| `pay_taxes` | Pay CHOAM taxes 1x (300–500cr) | Pay CHOAM taxes 3x (800–1,200cr) | Pay CHOAM taxes 5x (1,500–2,000cr) |

Generation rules:
- Always generate exactly 3 missions
- Ensure variety: no duplicate mission types in the same day's set
- Weighted selection: 40% easy, 35% medium, 25% hard (randomized)
- Target counts and rewards are random within the range

#### `trackMissionProgress(db, playerId, galaxyId, type, amount = 1): Promise<void>`
- Find active (unclaimed) mission of `type` for today
- Increment `current_count` by `amount`
- If `current_count >= target_count`, set `completed = 1`
- Do nothing if no matching active mission exists

#### `getTodaysMissions(db, playerId, galaxyId): Promise<Mission[]>`
- Calls `generateDailyMissions()` if needed
- Returns all missions for today with computed `progress` (0–100)

#### `claimMissionReward(db, playerId, galaxyId, missionId): Promise<{ success, reward, newCredits }>`
- Verify mission exists, is completed, and unclaimed
- Update `claimed = 1`, `claimed_at = now`
- Add `reward_credits` to `player_ships.credits`
- Return new credit balance

#### `rerollMission(db, playerId, galaxyId, missionId): Promise<{ success, newMission, cost }>`
- Verify mission is uncompleted and unclaimed
- Calculate cost: `max(500, floor(reward_credits * 0.5))`
- Deduct cost from `player_ships.credits`
- Delete old mission, generate a replacement of the same difficulty tier
- Return new mission

**Types:**
```typescript
interface Mission {
  id: number;
  type: string;
  targetCount: number;
  currentCount: number;
  rewardCredits: number;
  completed: boolean;
  claimed: boolean;
  progress: number; // 0–100
}
```

### Task 3: Hook mission progress into action endpoints

Add `trackMissionProgress()` calls at the **success point** of each relevant handler:

#### `cloud/src/routes/action.ts` — `handleCombat()`
After `result.won === true` (NPC killed):
```typescript
await trackMissionProgress(db, auth.playerId, galaxyId, 'kill_npcs', 1);
```

#### `cloud/src/routes/action.ts` — `handleTrade()`
After successful trade that results in **net profit** (sell price > buy price), track profit amount:
```typescript
// After updating ship credits/cargo:
if (action === 'sell' && profit > 0) {
  await trackMissionProgress(db, auth.playerId, galaxyId, 'trade_credits', profit);
}
```
> **Note:** Trade profit tracking requires computing profit per commodity. If this is too complex, track total sell value instead (simpler, still rewarding active traders).

**Decision:** Track **total credits earned from selling** (not net profit). Simpler, still requires active trading.

#### `cloud/src/routes/player.ts` — `handleMoveShip()`
After successful move (ship reaches new sector), track new sector visit:
```typescript
// Load visited_sectors_json, check if sectorId is new
const visited = JSON.parse(ship.visited_sectors_json ?? '[]');
if (!visited.includes(sectorId)) {
  visited.push(sectorId);
  await db.prepare('UPDATE player_ships SET visited_sectors_json = ? WHERE player_id = ? AND galaxy_id = ?')
    .bind(JSON.stringify(visited), auth.playerId, galaxyId).run();
  await trackMissionProgress(db, auth.playerId, galaxyId, 'visit_sectors', 1);
}
```

#### `cloud/src/routes/planets.ts` — `handleColonize()`
After successful colonization:
```typescript
await trackMissionProgress(db, auth.playerId, galaxyId, 'claim_planet', 1);
```

#### `cloud/src/routes/player.ts` — `handlePayTaxes()`
After successful tax payment:
```typescript
await trackMissionProgress(db, auth.playerId, galaxyId, 'pay_taxes', 1);
```

### Task 4: Create API route handlers
**File:** `cloud/src/routes/missions.ts`

#### `handleGetMissions(auth, galaxyId, db)` → `GET /api/player/missions?galaxyId=`
- Validate `galaxyId`
- Call `getTodaysMissions(db, auth.playerId, galaxyId)`
- Return `{ missions: [...], allClaimed: boolean }`

#### `handleClaimMission(auth, request, db)` → `POST /api/player/missions/claim`
- Body: `{ galaxyId, missionId }`
- Call `claimMissionReward(db, auth.playerId, galaxyId, missionId)`
- Return `{ success, reward, newCredits }`

#### `handleRerollMission(auth, request, db)` → `POST /api/player/missions/reroll`
- Body: `{ galaxyId, missionId }`
- Call `rerollMission(db, auth.playerId, galaxyId, missionId)`
- Return `{ success, newMission, cost }`
