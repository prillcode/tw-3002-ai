## Objective
Build the in-game mission panel so players can see daily bounties, track progress, claim rewards, and reroll missions without leaving the sector view.

---

## Context

**Server endpoints available (production):**
- `GET /api/player/missions?galaxyId=` — returns `{ missions: Mission[], allClaimed: boolean }`
- `POST /api/player/missions/claim` — body `{ galaxyId, missionId }` → `{ success, reward, newCredits }`
- `POST /api/player/missions/reroll` — body `{ galaxyId, missionId }` → `{ success, newMission, cost }`

**Mission shape:**
```typescript
interface Mission {
  id: number;
  type: string;          // 'kill_npcs' | 'trade_credits' | 'visit_sectors' | 'claim_planet' | 'pay_taxes'
  targetCount: number;
  currentCount: number;
  rewardCredits: number;
  completed: boolean;
  claimed: boolean;
  progress: number;      // 0–100
}
```

**UI patterns in the game client:**
- Keyboard shortcuts handled in `SectorView.vue` `handleKey()`
- Modals managed via `useUiStore()` (`activeModal`, `openModal()`, `closeModal()`)
- Modal content rendered inline in `SectorView.vue` with `v-if` conditions inside a `Teleport`
- Action buttons in left panel use `terminal-btn` class
- Color tokens: `text-terminal-cyan`, `text-terminal-green`, `text-terminal-yellow`, `text-terminal-red`, `text-terminal-muted`, `bg-void-800`, `bg-terminal-green/20`, etc.
- Full-page views (like Leaderboard) use `router.push()`

**Design decision:** Mission panel will be a **modal** (like Help/Nav), not a full-page view. Players need to check it frequently while playing. `B` key for "bounties".

### Task 2: Create `MissionPanel.vue` component
**File:** `web/game/src/components/MissionPanel.vue`

A self-contained component that fetches missions and renders them.

**Template structure:**
```vue
<div class="terminal-panel p-4 max-w-md w-full">
  <div class="flex items-center justify-between mb-4">
    <h2 class="font-mono font-bold text-terminal-yellow text-lg">📋 DAILY BOUNTIES</h2>
    <button @click="$emit('close')" class="terminal-btn text-xs">Esc</button>
  </div>

  <!-- Loading -->
  <div v-if="loading" class="text-terminal-cyan font-mono text-sm">Loading...</div>
  <div v-else-if="error" class="text-terminal-red font-mono text-sm">{{ error }}</div>

  <!-- Missions list -->
  <div v-else-if="missions.length === 0" class="text-terminal-muted font-mono text-sm">
    No active bounties.
  </div>

  <div v-else class="space-y-3">
    <div
      v-for="mission in missions"
      :key="mission.id"
      class="border rounded p-3 font-mono text-sm"
      :class="missionCardClass(mission)"
    >
      <div class="flex items-center justify-between mb-2">
        <span class="font-bold">{{ missionLabel(mission.type) }}</span>
        <span :class="mission.completed ? 'text-terminal-green' : 'text-terminal-yellow'">
          {{ mission.currentCount }}/{{ mission.targetCount }}
        </span>
      </div>

      <!-- Progress bar -->
      <div class="h-2 bg-void-800 rounded overflow-hidden mb-2">
        <div
          class="h-full transition-all duration-500"
          :class="mission.completed ? 'bg-terminal-green' : 'bg-terminal-cyan'"
          :style="{ width: mission.progress + '%' }"
        />
      </div>

      <div class="flex items-center justify-between">
        <span class="text-terminal-green">💰 {{ mission.rewardCredits.toLocaleString() }} cr</span>

        <!-- Claim button (completed & unclaimed) -->
        <button
          v-if="mission.completed && !mission.claimed"
          @click="claimMission(mission.id)"
          :disabled="claiming[mission.id]"
          class="terminal-btn text-xs bg-terminal-green/20 text-terminal-green border-terminal-green/50"
        >
          {{ claiming[mission.id] ? '...' : 'Claim' }}
        </button>

        <!-- Reroll button (uncompleted & unclaimed) -->
        <button
          v-else-if="!mission.completed && !mission.claimed"
          @click="rerollMission(mission.id)"
          :disabled="rerolling[mission.id]"
          class="terminal-btn text-xs text-terminal-muted hover:text-terminal-yellow"
        >
          {{ rerolling[mission.id] ? '...' : `Reroll (${rerollCost(mission)}cr)` }}
        </button>

        <!-- Claimed badge -->
        <span v-else-if="mission.claimed" class="text-terminal-green text-xs">✓ Claimed</span>
      </div>
    </div>
  </div>

  <!-- All claimed message -->
  <div v-if="allClaimed" class="mt-3 text-terminal-green font-mono text-xs text-center">
    🎉 All bounties claimed! New missions at 00:00 UTC.
  </div>
</div>
```

**Script requirements:**
- Fetch missions on mount
- Auto-refresh every 30 seconds (or after claim/reroll)
- `claimMission(id)` → POST to `/api/player/missions/claim`, then refresh ship credits from store
- `rerollMission(id)` → POST to `/api/player/missions/reroll`, then refresh missions
- `missionLabel(type)` → human-readable names:
  - `kill_npcs` → "Hunt Pirates"
  - `trade_credits` → "Trade Profits"
  - `visit_sectors` → "Explore Sectors"
  - `claim_planet` → "Colonize Planets"
  - `pay_taxes` → "Pay CHOAM Tariffs"
- `missionCardClass(mission)` → `border-terminal-green/50 bg-terminal-green/5` for completed, `border-void-600` for active, `border-terminal-muted/30 opacity-60` for claimed
- `rerollCost(mission)` → `Math.max(500, Math.floor(mission.rewardCredits * 0.5))`

### Task 4: Add mission progress toast notification
**File:** `web/game/src/views/SectorView.vue`

When any gameplay action returns (move, trade, combat, etc.), check if mission progress changed. The simplest approach: after any successful action that calls the API, also refresh missions.

Add a `refreshMissions()` function that fetches missions silently (no loading state) and checks if any mission just became completed. If so, show a toast.

**Toast implementation:**
Add a `missionToast` ref to SectorView:
```typescript
const missionToast = ref<{ message: string; visible: boolean } | null>(null);
```

After refreshing missions, check for newly completed:
```typescript
const newlyCompleted = newMissions.find(m => m.completed && !oldMissions.find(om => om.id === m.id)?.completed);
if (newlyCompleted) {
  missionToast.value = {
    message: `🎉 Bounty complete: ${missionLabel(newlyCompleted.type)}! Claim your reward.`,
    visible: true
  };
  setTimeout(() => { missionToast.value = null; }, 4000);
}
```

Render toast in template (absolute positioned, top-right):
```html
<div v-if="missionToast" class="fixed top-4 right-4 z-50">
  <div class="terminal-panel px-4 py-2 border-terminal-green/50 bg-terminal-green/10 text-terminal-green font-mono text-sm animate-pulse">
    {{ missionToast.message }}
  </div>
</div>
```

Call `refreshMissions()` after:
- Successful move (`handleMove` / `handleJump`)
- Successful trade (after returning from MarketView... actually, market is a separate route. Better: refresh on mount when coming back from market?)
- Successful combat
- Successful tax payment (StarDock)
- Successful planet colonization

**Simpler approach:** Just refresh missions every time the SectorView mounts or gains focus. Also refresh after modal close.
