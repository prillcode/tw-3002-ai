## Schema Changes

### `planets` table (new)
```sql
CREATE TABLE planets (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  galaxy_id INTEGER NOT NULL,
  sector_index INTEGER NOT NULL,
  owner_id INTEGER NOT NULL,
  class TEXT NOT NULL DEFAULT 'M', -- M | K | O | L | C | H | U
  colonists INTEGER NOT NULL DEFAULT 0,
  fuel INTEGER NOT NULL DEFAULT 0,
  organics INTEGER NOT NULL DEFAULT 0,
  equipment INTEGER NOT NULL DEFAULT 0,
  citadel_level INTEGER NOT NULL DEFAULT 0,
  sect_cannon_pct INTEGER NOT NULL DEFAULT 0, -- 0-100
  atmo_cannon_pct INTEGER NOT NULL DEFAULT 0, -- 0-100
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(galaxy_id, sector_index, owner_id)
);
```

### `sectors` column addition
```sql
ALTER TABLE sectors ADD COLUMN planet_count INTEGER NOT NULL DEFAULT 0;
```

> Note: `planet_count` tracks how many planets exist in a sector for U-class probability calculations.

## Planet Class Probabilities

Base probabilities (when sector is empty):
| Class | Probability |
|-------|-------------|
| M | 25% |
| K | 15% |
| O | 15% |
| L | 20% |
| C | 10% |
| H | 10% |
| U | 5% |

Sector crowding modifier: for each existing planet in the sector, shift 5% probability toward U-class (cap at 98%).

Example: sector with 3 planets → U-class probability = 5% + 15% = 20%, others reweighted.

## API Endpoints

### `POST /api/planets/create`
Launch a Genesis Torpedo in the current sector.

**Body:** `{ galaxyId, sectorId }`

**Validation:**
- Player must be in target sector
- Player has sufficient credits (80,000 cr)
- Sector has not exceeded max planet limit (5 per sector, matching TWGS default)
- Cannot create planets in FedSpace

**Response:** `{ success, planetId, class, sectorId, remainingCredits }`

### `GET /api/planets/sector?galaxyId=&sectorId=`
List planets in a sector.

**Response:** `{ planets: [{ id, ownerId, ownerName, class, colonists, citadelLevel }] }`

### `POST /api/planets/colonize`
Transport colonists from FedSpace to a planet.

**Body:** `{ galaxyId, planetId, quantity }`

**Validation:**
- Planet must exist and be owned by player
- Player must have sufficient fuel in ship (10 fuel/sector from FedSpace)
- Quantity must not exceed planet's max colonist capacity
- Player pays transport cost from ship credits

**Response:** `{ success, planetId, colonistsAdded, totalColonists, remainingFuel }`

### `GET /api/planets/:id`
Get full planet details.

**Response:** `{ id, class, colonists, fuel, organics, equipment, citadelLevel, productionRates, maxColonists }`

- **No shared planets.** A sector can have multiple planets from different owners.
