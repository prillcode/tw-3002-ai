# 05-04 — PvP Leaderboards & Stats

## Objective
Track and display PvP achievements across the galaxy.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- `GET /api/leaderboard` already returns net worth rankings
- No PvP-specific leaderboards exist

## Tasks
1. Extend `GET /api/leaderboard` with PvP stats:
   - Add `kills`, `deaths`, `kdr` to response
   - Add `?sort=kills` parameter
2. Create `GET /api/leaderboard/wanted`:
   - List wanted players by wanted level
   - Include bounty reward
3. Create `GET /api/player/stats`:
   - Return full player profile: kills, deaths, K/D, wanted status, net worth history
4. Update LeaderboardScreen (from TW-08):
   - Add tabs: Net Worth | Kills | Wanted
   - Color-code wanted players in red
5. Player profile card:
   - Show on sector entry if another player is present
   - Display: name, class, net worth, kills, wanted status

## Verification
- [ ] Leaderboard shows kills ranking
- [ ] Wanted players appear in red
- [ ] Player stats API returns correct K/D
   - Multiple players in same sector show profile cards

## Done When
- Leaderboard has PvP rankings
- Wanted players are visible
- Player stats are queryable
- Profile cards show in sector

## Notes
- K/D ratio = kills / max(deaths, 1) to avoid divide by zero
- Net worth history could be a simple array of past values
- Profile cards should be compact — don't clutter the sector view
