# 05-02 — Bounty & Reputation System

## Objective
Add consequences for indiscriminate PvP — wanted status and NPC bounty hunters.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- TW-03 NPC brain can already target players based on reputation
- No player reputation tracking exists yet

## Tasks
1. Add `wanted_kills` and `reputation_score` to `player_ships` table (or new table)
2. Track unprovoked PvP kills:
   - Kill a player with net worth > 10k = +1 wanted kill
   - Kill a new player = +3 wanted kills (harsh penalty)
3. Wanted status:
   - 3+ wanted kills in 24h = "WANTED"
   - Display ☠️ next to player name in sector
   - Decay: -1 wanted kill per 48h of no PvP
4. Bounty hunters:
   - NPC patrols actively hunt wanted players
   - When a wanted player enters a sector, patrols attack
   - Patrols get bonus damage vs wanted players
5. Bounty board:
   - `GET /api/leaderboard/wanted` — list wanted players
   - Display bounty reward (based on wanted level)

## Verification
- [ ] Player gets 3 kills → WANTED status
- [ ] NPC patrols attack wanted player on sight
- [ ] Wanted status decays after 48h of peace
- [ ] Bounty board lists wanted players
- [ ] Killing a new player adds 3 wanted kills

## Done When
- Wanted system creates real consequences
- NPCs hunt wanted players
- Bounty board is queryable
- System is fair (new player protection + harsh penalties for violating it)

## Notes
- Wanted status is per-galaxy, not global
- Patrols should call out wanted players: "Halt! You are wanted in this sector!"
- Consider adding a "bounty claimed" news item when an NPC kills a wanted player
