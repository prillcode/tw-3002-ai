# 05-02 — Bounty & Wanted System

## Objective
Track aggressive players and make them hunted by NPC patrols.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- TW-03 NPC brain handles patrol/bounty hunter behavior
- `pvp_kills` table tracks all kills with timestamps

## Schema

### `player_ships` columns (existing from 05-01)
- `kills`, `deaths`, `reputation`

### New: `wanted_players` view/query
Derived from `pvp_kills`:
```sql
SELECT attacker_id, COUNT(*) as kill_count
FROM pvp_kills
WHERE timestamp > datetime('now', '-24 hours')
GROUP BY attacker_id
HAVING COUNT(*) >= 3;
```

## API Endpoints

### `GET /api/bounty/board?galaxyId=`
List currently wanted players.

**Response:** `{ wanted: [{ playerId, displayName, killCount, sectorId, lastSeen }] }`

### `GET /api/bounty/status?galaxyId=`
Check if the authenticated player is wanted.

**Response:** `{ wanted: boolean, killCount: number, hoursRemaining: number|null }`

## Logic

**Wanted threshold:** 3+ unprovoked kills in 24 hours.
- Unprovoked = victim did not attack first (no prior aggression from victim in last hour)
- For NPC kills: always count as provoked (hunting NPCs is expected)
- For first pass: all player-caused kills count as unprovoked (simplification)

**Decay:** Wanted status drops 1 kill every 48 hours of no PvP activity.

**NPC patrol integration (TW-03):**
- Wanted players have a chance to spawn hostile patrol NPCs in their current sector
- Patrol strength scales with wanted level (more kills = tougher patrols)
- Patrol NPCs actively hunt the wanted player
