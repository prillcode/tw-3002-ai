# 05-05 — Protections & Balance

## Objective
Prevent griefing, protect newcomers, and give risk-averse players options.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- Protections are checked in `resolve-defeat` endpoint
- FedSpace sectors already exist in galaxy data

## Protection Systems

### New Player Protection
- Net worth < 10,000 credits OR account age < 24 hours = invulnerable
- Checked in `resolve-defeat` — abort if victim is protected
- Visual indicator: green shield icon next to player name

### FedSpace Protection
- Any sector with `danger = 'safe'` is FedSpace
- No PvP possible in FedSpace (no defeat events, no fighter deployment)
- Checked in `resolve-defeat` and `fighter/deploy`

### Self-Attack Prevention
- `resolve-defeat` aborts if attacker_id == victim_id

### Offline Vulnerability
- Offline players are vulnerable to defeat (no fighter deployment yet — TW-06 adds offline defense)
- Exception: in FedSpace or under new-player protection

## Insurance System

### StarDock Insurance
- Purchase at StarDock for 5% of current net worth
- Duration: 7 days (or until death)
- Effect: reduces death penalty from 25% credits lost → 5% credits lost
- One insurance policy per ship at a time

### Schema
```sql
ALTER TABLE player_ships ADD COLUMN insurance_expires DATETIME;
ALTER TABLE player_ships ADD COLUMN insurance_active INTEGER DEFAULT 0;
```

### API
- `POST /api/insurance/buy` — purchase insurance
- `GET /api/insurance/status` — check if insured
