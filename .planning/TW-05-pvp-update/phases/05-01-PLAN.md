# 05-01 — Loot, Respawn & Kill Tracking

## Objective
Implement the PvP death pipeline: when a player is defeated (by NPC, fighter, or future PvP), calculate loot, respawn them, and record the kill.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- `pvp_kills` table exists in D1 schema but is unused
- Combat resolver already works for NPC combat — we hook into its result
- **No ship-to-ship dueling.** This pipeline is triggered by any defeat event.

## Schema

### Existing `pvp_kills` table
```sql
CREATE TABLE pvp_kills (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  galaxy_id INTEGER NOT NULL,
  attacker_id INTEGER NOT NULL,
  victim_id INTEGER NOT NULL,
  sector_id INTEGER NOT NULL,
  loot_credits INTEGER,
  loot_cargo_json TEXT,
  timestamp DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## API Endpoints

### `POST /api/combat/resolve-defeat`
Called by the combat system when a player is defeated.

**Body:** `{ galaxyId, victimId, attackerId?, sectorId, cause: 'npc' | 'fighter' | 'pvp' }`

**Logic:**
1. Load victim ship data
2. Check protections: if victim is new player (<10k net worth or <24h old), abort (they cannot be defeated)
3. Calculate loot:
   - `lootCredits = floor(victim.credits * 0.25)` — 25% for first pass
   - `lootCargo = {}` — 10% of each cargo type, rounded down
4. Update victim:
   - `credits -= lootCredits`
   - Remove looted cargo
   - `deaths += 1`
   - `current_sector = nearest FedSpace sector`
   - `hull = maxHull`
   - `shield = maxShield`
5. If attacker exists:
   - `attacker.credits += lootCredits`
   - Add looted cargo to attacker
   - `attacker.kills += 1`
6. Insert into `pvp_kills`
7. Insert news item
8. Return: `{ respawned, loot, fedSpaceSector }`

### `GET /api/player/stats?galaxyId=`
Return PvP stats for the authenticated player.

**Response:** `{ kills, deaths, kdr, reputation, wanted }`
