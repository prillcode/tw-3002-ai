# 05-01 — PvP Combat Foundation

## Objective
Players can attack each other in shared cloud galaxies via ship-to-ship dueling.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- `pvp_kills` table exists in D1 schema but is unused
- Combat resolver already works for NPC combat — reuse it
- **This is a simplified first pass.** True TW2002 PvP is fighter-based sector control (TW-06).

## Tasks
1. Create `POST /api/action/pvp/attack` endpoint:
   - Validate attacker is authenticated
   - Validate target player is in same sector
   - Check protections: FedSpace, new player (<10k net worth or <24h old), self-attack
   - Return target player data (name, hull, shield, ship class)
2. Create `POST /api/action/pvp/resolve` endpoint:
   - Run combat resolver (same as NPC combat)
   - Calculate loot: **25% credits** + 10% cargo (random commodity)
   - Update attacker: +loot, +kills
   - Update victim: respawn in FedSpace, 90% credits, cargo lost, +deaths
   - Insert news item: "Attacker destroyed Victim in Sector X"
   - Insert into `pvp_kills` table
3. Create `GET /api/action/pvp/status` endpoint:
   - Return whether PvP is possible in current sector
   - List other players in same sector (excluding protected ones)
4. Add PvP option to web client SectorView:
   - If other players present, show "Attack Player" option in NPC list or Actions
   - Select target player, then action (attack/flee/bribe)
5. Protection systems:
   - New player invulnerability check (net worth < 10k or account < 24h)
   - FedSpace block
   - Self-attack prevention
   - Offline players are vulnerable (no fighter deployment yet — TW-06 adds true offline defense)

## Verification
- [ ] Two test accounts in same sector
- [ ] Account A attacks Account B
- [ ] Combat resolves with damage numbers
- [ ] Winner gets loot, loser respawns
- [ ] News item generated
- [ ] FedSpace blocks PvP
- [ ] New player (<10k) is invulnerable

## Done When
- Players can attack each other
- Combat is fair (same resolver)
- Protections work
- News ticker shows PvP events

## Notes
- **Loot is intentionally conservative (25% credits).** TW2002's 50% is harsh without defensive layers (fighters, insurance, cloaking). We'll raise this after Phase 5 insurance and TW-06 fighter deployment.
- Consider adding a "warning" before attacking ("This player is new — are you sure?")
- **Combat is simultaneous** for simplicity. Future: ordered combat sequence per `../lore-reference/strategy/combat.md` (empty ships → player ships by sign-on order → sector fighters).
- Offline attacks: for first pass, process immediately. In TW-06, deployed fighters will auto-defend offline players instead.
