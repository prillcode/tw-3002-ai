# 05-03 — News & Notifications

## Objective
Inform players about PvP events through news and notifications.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- News system already exists (`news` table, `/api/news` endpoint)
- `pvp_kills` table provides event data

## API Endpoints

### Existing: `GET /api/news?galaxyId=&limit=`
Already returns news items. We add PvP news generation.

### New: `POST /api/notifications/digest`
Return a summary of events since player's last login.

**Body:** `{ galaxyId }`

**Logic:**
1. Get player's `last_login_at` from `players` table
2. Query `pvp_kills` where victim_id = player OR attacker_id = player, timestamp > last_login
3. Query `news` where galaxy_id = galaxyId, timestamp > last_login
4. Return summary

**Response:**
```json
{
  "attacksOnYou": 2,
  "deaths": 1,
  "kills": 0,
  "news": [...],
  "bountyChanges": "placed"
}
```

### New: Discord webhook
- Player stores `discord_webhook_url` in settings (opt-in)
- On PvP event involving player, POST to webhook

## News Generation Triggers

| Event | News Headline |
|-------|---------------|
| Player killed by player | "DarkReaver destroyed prillcode in Sector 42" |
| Player killed by NPC | "prillcode was destroyed by Pirate Raider in Sector 7" |
| Wanted status placed | "DarkReaver is now WANTED — 5 kills in 24 hours" |
| Wanted status cleared | "DarkReaver's bounty has expired" |
