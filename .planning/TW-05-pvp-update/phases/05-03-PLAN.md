# 05-03 — PvP Notifications

## Objective
Players know when they've been attacked or killed.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- Discord webhook URL is already a secret in the Worker
- No notification system exists yet

## Tasks
1. Discord webhook integration:
   - Read `DISCORD_WEBHOOK_URL` from env secrets
   - Send webhook on PvP events: attacked, destroyed, bounty placed, bounty cleared
2. In-game kill log:
   - Add `kill_log` JSON column to `player_ships` or new table
   - Store: attacker, victim, sector, credits looted, timestamp
3. Digest mode:
   - On login, check for offline attacks
   - Show summary: "While you were away: 2 attacks, 1 death, lost 5,000 cr"
4. Notification payload format:
   ```
   ⚔️ PvP Alert
   Your ship was destroyed by DarkReaver in Sector 42.
   Loot lost: 12,000 credits, 15 ore
   Respawn: Sol Prime (FedSpace)
   ```
5. Opt-in/opt-out:
   - Default: Discord notifications enabled if webhook configured
   - Player can disable in settings (future)

## Verification
- [ ] Set Discord webhook URL via `wrangler secret put`
   - Attack another player → webhook fires
   - Get destroyed → webhook fires
   - Digest shown on login after offline death

## Done When
- Discord webhooks fire on PvP events
- In-game digest shows offline activity
- Notifications are informative and actionable

## Notes
- Don't spam webhooks — one per event, not per round
- Test webhook with a private Discord channel first
- Consider email notifications as a fallback (reuse auth email)
