# 06-03 — NPC Tick Endpoint

## Objective
Make NPCs move, trade, and fight in the cloud galaxy so it feels alive.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- NPCs are seeded but never update after initial insert
- The engine's `tickNPCs()` uses `process` and can't run in the Worker
- Need a lightweight rule-based system that runs in the Worker runtime

## Tasks
1. Create `POST /api/npc/tick` endpoint (or Cron Trigger)
2. Implement rule-based NPC decision loop:
   - Fetch all active NPCs for the galaxy
   - For each NPC, decide action based on persona type:
     - **Trader:** Move toward nearest port. If at port, buy cheapest / sell most expensive.
     - **Raider:** Move toward dangerous sector. If player or weak NPC present, attack.
     - **Patrol:** Move toward FedSpace edge. If raider present, attack.
3. Update `npcs` table: `current_sector`, `credits`, `cargo_json`, `memory_json`
4. Generate news items for significant events:
   - NPC killed another NPC
   - NPC destroyed a player ship
   - Large trade transaction
5. Return summary: `{ npcsProcessed, actionsTaken, newsGenerated }`
6. Set up Cron Trigger in `wrangler.toml` to run every 5 minutes
7. Update `CloudSectorScreen` to refresh sector NPCs on entry

## Verification
- [ ] `curl POST /api/npc/tick` returns a summary
- [ ] After tick, `curl /api/galaxy/1/sector?id=X` shows different NPC positions
- [ ] News table has new entries after ticks
- [ ] Cron Trigger fires every 5 minutes (check Wrangler logs)
- [ ] NPCs don't disappear or duplicate

## Done When
- NPCs move between sectors every 5 minutes
- NPCs trade at ports
- NPCs fight each other
- News ticker reflects real events
- System is deterministic and doesn't corrupt data

## Notes
- **No LLM** — pure rule-based for reliability and cost
- Memory/JSON fields should store last action for continuity
- Consider rate limiting the tick endpoint (admin-only or cron-only)
- The tick should be idempotent — running twice shouldn't break anything
- Test with a small galaxy (100 sectors) before scaling to 1000
