# 06-01 — Trade Prices in Cloud Mode

## Objective
Make trade prices visible in the cloud mode trade overlay so players can make informed buy/sell decisions.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- Cloud API returns `port_inventory_json` (e.g., `{"ore":{"price":102,"supply":1135}}`)
- `CloudSectorScreen` has a trade overlay but doesn't parse this data
- `cloudGetSector()` already returns the raw JSON

## Tasks
1. Parse `port_inventory_json` from `cloudGetSector()` response in `loadSector()`
2. Store parsed inventory on the sector object (or in component state)
3. In the trade overlay, display current price for each commodity
4. Show "profit" indicator if player has cargo of that commodity (current price vs implied buy price)
5. Update `handleTrade()` to use the displayed prices for validation feedback

## Verification
- [ ] `curl` the sector endpoint and verify `port_inventory_json` exists
- [ ] Launch CLI, enter cloud mode, warp to a port sector
- [ ] Press `M` — trade overlay shows prices for ore/organics/equipment
- [ ] Prices match the raw JSON from the API

## Done When
- Trade overlay displays buy/sell prices for all 3 commodities
- Prices are accurate to the D1 data
- UI doesn't break on sectors without ports

## Notes
- The trade overlay currently shows quantity and cargo counts only
- Prices should update on every sector load (fresh API call)
- No server changes needed — purely client-side parsing
